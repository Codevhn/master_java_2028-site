3.7: Verificación correcta de la instalación de Java

Después de instalar el JDK y configurar las variables de entorno, es esencial verificar que todo está configurado correctamente antes de proceder con desarrollo real. Una verificación exhaustiva en este punto previene descubrir problemas días o semanas después cuando se está en medio de un proyecto y algo inesperadamente falla. Este subtema proporciona un checklist completo de verificaciones que confirman que la instalación de Java está correcta y funcional.

Nivel 1: Verificaciones básicas de línea de comandos

Abrir una nueva ventana de terminal según el sistema operativo:

  • Windows: cmd o PowerShell (Win + R → escribir cmd)
  • macOS: Terminal (Applications → Utilities → Terminal)
  • Linux: Terminal (Ctrl + Alt + T en la mayoría de distribuciones)

Si se acaban de configurar las variables de entorno, abrir nueva ventana de terminal. Las ventanas abiertas antes de la configuración no tendrán las variables actualizadas.

Verificación 1.1: Versión de Java Runtime

  java -version

Salida esperada (ejemplo para Java 21):

  openjdk version "21.0.1" 2023-10-17 LTS
OpenJDK Runtime Environment Temurin-21.0.1+12 (build 21.0.1+12-LTS)
OpenJDK 64-Bit Server VM Temurin-21.0.1+12 (build 21.0.1+12-LTS, mixed mode, sharing)

Qué verificar: el comando ejecuta sin error, la versión mostrada coincide con la instalada, muestra "64-Bit", y muestra "LTS" si se instaló versión LTS.

Problemas comunes:

Error Causa probable Solución
'java' is not recognized... PATH no configurado Revisar configuración de PATH en subtema 3.6
Versión incorrecta mostrada Otra instalación Java en PATH antes Ajustar orden en PATH, verificar JAVA_HOME
Unable to locate JRE Instalación corrupta Reinstalar JDK

Verificación 1.2: Versión del compilador

  javac -version

  javac 21.0.1

La versión debe coincidir exactamente con java -version. Si javac falla pero java funciona, se instaló el JRE en lugar del JDK completo — reinstalar el JDK.

Verificación 1.3: Otras herramientas del JDK

  # Herramienta de empaquetado
jar --version

# Generador de documentación
javadoc --version

# Desensamblador de bytecode
javap -version

Cada comando debe mostrar la versión coincidiendo con el Java instalado. Si alguno falla, el PATH no está configurado correctamente o la instalación está incompleta.

Nivel 2: Verificación de variables de entorno

Verificación 2.1: JAVA_HOME está configurada

  # Windows (CMD)
echo %JAVA_HOME%

# Windows (PowerShell)
echo $env:JAVA_HOME

# macOS/Linux
echo $JAVA_HOME

Salidas esperadas:

  # Windows
C:\Program Files\Eclipse Adoptium\jdk-21.0.1.12-hotspot

# macOS
/Library/Java/JavaVirtualMachines/temurin-21.jdk/Contents/Home

# Linux
/opt/jdk-21

Verificar que no está vacía, que apunta al directorio raíz del JDK (no al subdirectorio bin), y que la ruta existe:

  # Windows
dir "%JAVA_HOME%"

# macOS/Linux
ls $JAVA_HOME

Debe mostrar subdirectorios: bin, lib, include, conf, jmods.

Verificación 2.2: PATH contiene bin del JDK

  # Windows
echo %PATH%

# macOS/Linux (filtrar solo entradas Java)
echo $PATH | tr ':' '\n' | grep java

Debe contener una entrada con JAVA_HOME/bin o la ruta explícita al bin del JDK.

Verificación 2.3: Ubicación de ejecutables

  # macOS/Linux
which java
which javac

# Windows
where java
where javac

Salida esperada:

  /Library/Java/JavaVirtualMachines/temurin-21.jdk/Contents/Home/bin/java
/Library/Java/JavaVirtualMachines/temurin-21.jdk/Contents/Home/bin/javac

Si en macOS muestra /usr/bin/java (wrapper del sistema), verificar que apunta a la versión correcta con:

  /usr/libexec/java_home -V

Nivel 3: Prueba de compilación y ejecución

La prueba definitiva es compilar y ejecutar un programa Java simple.

Paso 1: Crear directorio de prueba:

  mkdir ~/java-test
cd ~/java-test

Paso 2: Crear archivo HolaMundo.java:

  public class HolaMundo {
    public static void main(String[] args) {
        System.out.println("¡Hola, Mundo!");
        System.out.println("Java está correctamente instalado.");
        System.out.println("Versión de Java: " + System.getProperty("java.version"));
        System.out.println("Vendor: " + System.getProperty("java.vendor"));
        System.out.println("JAVA_HOME: " + System.getProperty("java.home"));
    }
}

Crear el archivo con un editor de texto:

  # Windows
notepad HolaMundo.java

# macOS/Linux
nano HolaMundo.java
# Pegar código, Ctrl+O para guardar, Enter, Ctrl+X para salir

Paso 3: Compilar:

  javac HolaMundo.java

Sin salida significa éxito. Verificar que se creó HolaMundo.class:

  ls -l        # macOS/Linux
dir          # Windows

Paso 4: Ejecutar:

  java HolaMundo

  ¡Hola, Mundo!
Java está correctamente instalado.
Versión de Java: 21.0.1
Vendor: Eclipse Adoptium
JAVA_HOME: /Library/Java/JavaVirtualMachines/temurin-21.jdk/Contents/Home

Errores comunes en ejecución:

Error Causa Solución
Error: Could not find or load main class HolaMundo Nombre de clase no coincide con nombre de archivo Verificar que el archivo se llama exactamente HolaMundo.java
NoClassDefFoundError Ejecutando desde directorio incorrecto Asegurar estar en el directorio que contiene HolaMundo.class
javac: command not found javac no está en PATH Revisar configuración de PATH

Paso 5: Limpiar archivos de prueba:

  # macOS/Linux
cd ~
rm -rf ~/java-test

# Windows
cd %USERPROFILE%
rmdir /s java-test

Nivel 4: Verificación de información detallada del sistema

Para diagnóstico avanzado, crear y ejecutar VerificarJava.java:

  public class VerificarJava {
    public static void main(String[] args) {
        System.out.println("=== Verificación de Instalación Java ===\n");

        System.out.println("Versión Java: " + System.getProperty("java.version"));
        System.out.println("Vendor: " + System.getProperty("java.vendor"));
        System.out.println("Vendor URL: " + System.getProperty("java.vendor.url"));

        System.out.println("\nJAVA_HOME: " + System.getProperty("java.home"));
        System.out.println("java.class.path: " + System.getProperty("java.class.path"));

        System.out.println("\nSistema Operativo: " + System.getProperty("os.name"));
        System.out.println("Versión OS: " + System.getProperty("os.version"));
        System.out.println("Arquitectura: " + System.getProperty("os.arch"));

        Runtime runtime = Runtime.getRuntime();
        System.out.println("\nMemoria Máxima JVM: " + (runtime.maxMemory() / 1024 / 1024) + " MB");
        System.out.println("Memoria Total JVM: " + (runtime.totalMemory() / 1024 / 1024) + " MB");

        System.out.println("\nUsuario: " + System.getProperty("user.name"));
        System.out.println("Directorio home: " + System.getProperty("user.home"));
        System.out.println("Directorio actual: " + System.getProperty("user.dir"));

        System.out.println("\n=== Verificación Completada ===");
    }
}

  javac VerificarJava.java
java VerificarJava

# Guardar salida para documentación
java VerificarJava > verificacion-java.txt

Nivel 5: Checklist final de verificación

Básico:

  • java -version muestra versión correcta
  • javac -version muestra versión correcta
  • jar --version funciona
  • JAVA_HOME está configurada correctamente
  • PATH contiene bin del JDK
  • which java / where java apunta a ubicación correcta

Funcional:

  • Programa simple compila exitosamente con javac
  • Programa compilado ejecuta exitosamente con java
  • Salida muestra versión de Java correcta

Avanzado:

  • Múltiples herramientas JDK funcionan (javadoc, javap, jar)
  • Propiedades del sistema muestran valores correctos
  • Sin conflictos con instalaciones previas de Java

Documentación:

  • Versión de JDK instalada documentada
  • Distribución usada documentada (Temurin, Corretto, etc.)
  • Ubicación de instalación anotada
  • Valores de JAVA_HOME y PATH registrados

Resolución de problemas comunes

IDE no encuentra JDK a pesar de que todo parece correcto. Solución: reiniciar el IDE después de configurar las variables de entorno. En la configuración del IDE, especificar la ruta JAVA_HOME manualmente y verificar que está buscando JDK, no JRE.

Versión incorrecta ejecuta a pesar de configuración correcta. Solución: verificar que no hay otra instalación de Java en una ubicación con precedencia. En Windows, verificar que C:\Windows\System32 no contiene java.exe. En macOS, usar /usr/libexec/java_home -V para ver todas las versiones. Asegurar que JAVA_HOME/bin está al inicio del PATH.

Funciona en una terminal pero no en otra. Solución: las variables de entorno son específicas a la sesión de terminal. Asegurar que la configuración está en el archivo correcto (.bashrc vs .bash_profile vs .zshrc), luego ejecutar source ~/.bashrc (o el archivo apropiado) en las terminales afectadas.

Una vez que todas las verificaciones pasan exitosamente, la instalación de Java está completa y correcta. El entorno está listo para desarrollo real, instalación de IDEs, configuración de sistemas de construcción, y trabajo en proyectos Java.

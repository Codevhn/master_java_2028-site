4.7 Bloques de código y uso de llaves {}

Las llaves { } son elementos fundamentales en Java que delimitan y organizan el código en bloques lógicos. No son simples símbolos decorativos sino marcadores estructurales que definen el alcance y la jerarquía del código. Comprender qué son los bloques de código, cómo funcionan las llaves, por qué deben estar balanceadas, y cómo usar la indentación para hacer el código legible es esencial para escribir programas Java correctos y bien organizados. Este subtema explora el concepto de bloques de código y el uso correcto de llaves en la estructura de programas Java.

¿Qué son las llaves { }?

Las llaves son símbolos que agrupan instrucciones formando un bloque de código.

Símbolo de apertura: { (abre un bloque)
Símbolo de cierre: } (cierra un bloque)

Regla fundamental: Por cada { debe haber exactamente un } correspondiente.

Analogía simple:

Las llaves son como paréntesis en matemáticas:

  • (2 + 3) = Los paréntesis agrupan la operación
  • { instrucciones } = Las llaves agrupan el código

O como las tapas de una caja:

  • { = Tapa superior (abre la caja)
  • Contenido = Lo que está dentro
  • } = Tapa inferior (cierra la caja)

Bloques de código en Java

Un bloque de código es todo lo que está entre una llave de apertura { y su correspondiente llave de cierre }.

Ejemplo básico:

{
    // Este es un bloque de código
    double precio = 100.0;
    double descuento = 10.0;
    double total = precio - descuento;
}

Todo lo que está entre { y } pertenece al mismo bloque.

Bloques de código en la estructura de un programa

En un programa Java típico hay múltiples niveles de bloques anidados:

public class MiPrograma { // ← Bloque de la clase comienza

    public static void main(String[] args) { // ← Bloque del método comienza

        double precioProducto = 250.00;
        double descuento = 25.00;
        double precioFinal = precioProducto - descuento;

        System.out.println("Precio final: Lps " + precioFinal);

    } // ← Bloque del método termina

} // ← Bloque de la clase termina

Visualización de bloques:

Bloque de clase: {
    Bloque de método: {
        Instrucciones del método
    }
}

Niveles de anidamiento (bloques dentro de bloques)

Los bloques pueden contener otros bloques, formando una jerarquía:

public class Ejemplo { // Nivel 1: Bloque de clase

    public static void main(String[] args) { // Nivel 2: Bloque de método

        double saldo = 1000.0;
        double retiro = 150.0;
        double nuevoSaldo = saldo - retiro;

        System.out.println("Saldo: Lps " + nuevoSaldo);

    } // Cierra nivel 2

} // Cierra nivel 1

Estructura jerárquica:

{ Clase
    { Método main
        Instrucción 1
        Instrucción 2
        Instrucción 3
    } Fin método
} Fin clase

Ejemplo: Calculadora de descuento con bloques claros

public class CalculadoraDescuento { // { Bloque de clase

    public static void main(String[] args) { // { Bloque de main

        // Datos del producto
        double precioOriginal = 850.00;
        double porcentajeDescuento = 15.0;

        // Calcular descuento
        double montoDescuento = precioOriginal * porcentajeDescuento / 100.0;

        // Calcular precio final
        double precioFinal = precioOriginal - montoDescuento;

        // Mostrar resultados
        System.out.println("===== DESCUENTO =====");
        System.out.println("Precio original: Lps " + precioOriginal);
        System.out.println("Descuento: Lps " + montoDescuento);
        System.out.println("Precio final: Lps " + precioFinal);

    } // } Fin bloque main

} // } Fin bloque clase

Conteo de llaves:

  • 2 llaves de apertura {
  • 2 llaves de cierre }
  • ✓ Balanceadas correctamente

Regla de balanceo de llaves

Cada { DEBE tener su correspondiente }

Correcto (balanceado):

{
    double valor = 100.0;
}

Incorrecto (desbalanceado):

{
    double valor = 100.0;
// ✗ Falta }

Error de compilación:

error: reached end of file while parsing
}
^

Incorrecto (llave de cierre sin apertura):

double valor = 100.0;
} // ✗ } sin {

Error de compilación:

error: illegal start of expression

Indentación: Haciendo el código legible

La indentación (sangría) es el espacio al inicio de cada línea que muestra visualmente la jerarquía de bloques.

Sin indentación (difícil de leer):

public class Ejemplo {
public static void main(String[] args) {
double precio = 100.0;
double descuento = 10.0;
double total = precio - descuento;
System.out.println("Total: " + total);
}
}

Con indentación correcta (fácil de leer):

public class Ejemplo {
    public static void main(String[] args) {
        double precio = 100.0;
        double descuento = 10.0;
        double total = precio - descuento;
        System.out.println("Total: " + total);
    }
}

La indentación NO afecta la compilación, pero hace el código mucho más legible.

Reglas de indentación

Regla 1: Cada nivel de bloque aumenta la indentación

public class MiClase {                      // Nivel 0
    public static void main(...) {          // Nivel 1 (4 espacios)
        double x = 10.0;                   // Nivel 2 (8 espacios)
        double y = 20.0;                   // Nivel 2 (8 espacios)
    }                                       // Vuelve a nivel 1
}                                           // Vuelve a nivel 0

Regla 2: Usar consistentemente 4 espacios por nivel

public class Ejemplo {
    public static void main(String[] args) {
        // ← 4 espacios desde el borde
        double valor = 100.0;
        // ← 4 espacios desde el borde
    }
}

Regla 3: Las llaves de cierre alinean con el inicio del bloque

public class Ejemplo {
    public static void main(String[] args) {
        double valor = 100.0;
    } // ← Alineado con "public static void main"
} // ← Alineado con "public class Ejemplo"

Ejemplo con indentación correcta

public class CalculadoraSalario {

    public static void main(String[] args) {

        // Datos del empleado
        double horasTrabajadas = 45.0;
        double tarifaPorHora = 125.00;

        // Calcular salario base
        double salarioBase = horasTrabajadas * tarifaPorHora;

        // Calcular bono (10% del salario)
        double porcentajeBono = 10.0;
        double montoBono = salarioBase * porcentajeBono / 100.0;

        // Calcular salario total
        double salarioTotal = salarioBase + montoBono;

        // Mostrar desglose
        System.out.println("===== CÁLCULO DE SALARIO =====");
        System.out.println("Horas trabajadas: " + horasTrabajadas);
        System.out.println("Tarifa por hora: Lps " + tarifaPorHora);
        System.out.println("------------------------------");
        System.out.println("Salario base: Lps " + salarioBase);
        System.out.println("Bono (10%): Lps " + montoBono);
        System.out.println("==============================");
        System.out.println("SALARIO TOTAL: Lps " + salarioTotal);

    }

}

Observaciones:

  • Las instrucciones dentro de main están indentadas 4 espacios
  • Las llaves } de cierre alinean con el inicio de sus bloques
  • El código es fácil de seguir visualmente

Errores comunes con llaves

Error 1: Olvidar llave de cierre

public class Ejemplo {
    public static void main(String[] args) {
        double valor = 100.0;
        System.out.println(valor);
    }
// ✗ Falta cerrar la clase

Error de compilación:

error: reached end of file while parsing

Solución: Agregar } al final

Error 2: Llaves extras

public class Ejemplo {
    public static void main(String[] args) {
        double valor = 100.0;
    }
}
} // ✗ Llave extra

Error de compilación:

error: class, interface, or enum expected

Error 3: Llaves mal ubicadas

public class Ejemplo
{ // ✗ Estilo incorrecto (aunque compila)
    public static void main(String[] args)
    { // ✗ Estilo incorrecto
        double valor = 100.0;
    }
}

Aunque esto compila, NO es el estilo estándar de Java.

Estilo correcto de Java:

public class Ejemplo { // ✓ Llave en misma línea
    public static void main(String[] args) { // ✓ Llave en misma línea
        double valor = 100.0;
    }
}

Convención de posición de llaves en Java

Estilo Java estándar (llaves en misma línea):

public class MiClase {
    public static void main(String[] args) {
        // Código
    }
}

NO usar (estilo Allman, común en C#):

public class MiClase
{
    public static void main(String[] args)
    {
        // Código
    }
}

Recomendación: Siempre poner la llave de apertura { en la misma línea que la declaración.

Verificar balanceo de llaves

Técnica manual:

Cuente las llaves en su programa:

public class Ejemplo { // 1 {
    public static void main(...) { // 2 {
        double x = 10.0;
    } // 1 }
} // 2 }

Total: 2 llaves abren, 2 llaves cierran → ✓ Balanceadas

Los IDEs ayudan automáticamente:

Los editores modernos:

  • Resaltan la llave correspondiente cuando coloca el cursor
  • Indentan automáticamente
  • Muestran errores si las llaves están desbalanceadas

Ejemplo: Sistema de cálculo de factura

public class SistemaFactura {

    public static void main(String[] args) {

        // Bloque principal del programa

        System.out.println("========== FACTURA ==========");

        // Datos de productos
        double precioProducto1 = 450.00;
        double precioProducto2 = 320.00;
        double precioProducto3 = 175.50;

        // Calcular subtotal
        double subtotal = precioProducto1 + precioProducto2 + precioProducto3;

        // Calcular impuesto (15%)
        double tasaImpuesto = 15.0;
        double montoImpuesto = subtotal * tasaImpuesto / 100.0;

        // Calcular total
        double totalFactura = subtotal + montoImpuesto;

        // Mostrar desglose
        System.out.println("Producto 1: Lps " + precioProducto1);
        System.out.println("Producto 2: Lps " + precioProducto2);
        System.out.println("Producto 3: Lps " + precioProducto3);
        System.out.println("-----------------------------");
        System.out.println("Subtotal: Lps " + subtotal);
        System.out.println("Impuesto (15%): Lps " + montoImpuesto);
        System.out.println("=============================");
        System.out.println("TOTAL: Lps " + totalFactura);

    } // Cierra bloque de main

} // Cierra bloque de clase

Verificación:

  • ✓ Llaves balanceadas (2 aperturas, 2 cierres)
  • ✓ Indentación correcta (4 espacios por nivel)
  • ✓ Llaves de cierre alineadas
  • ✓ Código legible y organizado

Espacios en blanco dentro de bloques

Puede usar líneas en blanco para organizar visualmente:

public class Ejemplo {

    public static void main(String[] args) {

        // Sección 1: Entrada de datos
        double precioBase = 500.0;
        double descuento = 50.0;

        // Sección 2: Cálculos
        double precioFinal = precioBase - descuento;

        // Sección 3: Salida
        System.out.println("Precio final: Lps " + precioFinal);

    }

}

Las líneas en blanco no afectan la ejecución, solo mejoran legibilidad.

Buenas prácticas con bloques y llaves

1. Siempre indentar consistentemente

Use 4 espacios por nivel de indentación.

2. Alinear llaves de cierre con el inicio del bloque

public class Ejemplo {
    public static void main(String[] args) {
        // Código
    } // ← Alineado con "public static void main"
} // ← Alineado con "public class"

3. No poner múltiples instrucciones en una línea

// ✗ Difícil de leer
double x = 10; double y = 20; double z = x + y;

// ✓ Fácil de leer
double x = 10;
double y = 20;
double z = x + y;

4. Usar líneas en blanco para separar secciones lógicas

// Datos
double precio = 100.0;

// Cálculos
double descuento = precio * 0.1;

// Resultado
System.out.println(descuento);

Ejemplo final: Control de inventario

public class ControlInventario {

    public static void main(String[] args) {

        System.out.println("===== CONTROL DE INVENTARIO =====");
        System.out.println();

        // Inventario inicial
        int stockInicial = 150;

        // Movimientos del día
        int entradasMañana = 30;
        int salidasMañana = 45;
        int entradasTarde = 20;
        int salidasTarde = 35;

        // Calcular total de entradas y salidas
        int totalEntradas = entradasMañana + entradasTarde;
        int totalSalidas = salidasMañana + salidasTarde;

        // Calcular stock final
        int stockFinal = stockInicial + totalEntradas - totalSalidas;

        // Mostrar reporte
        System.out.println("Stock inicial: " + stockInicial + " unidades");
        System.out.println("----------------------------------");
        System.out.println("Entradas mañana: +" + entradasMañana);
        System.out.println("Salidas mañana: -" + salidasMañana);
        System.out.println("Entradas tarde: +" + entradasTarde);
        System.out.println("Salidas tarde: -" + salidasTarde);
        System.out.println("----------------------------------");
        System.out.println("Total entradas: +" + totalEntradas);
        System.out.println("Total salidas: -" + totalSalidas);
        System.out.println("==================================");
        System.out.println("Stock final: " + stockFinal + " unidades");

    }

}

Características del código:

  • ✓ Llaves balanceadas perfectamente
  • ✓ Indentación consistente de 4 espacios
  • ✓ Líneas en blanco separan secciones lógicas
  • ✓ Comentarios indican propósito de cada sección
  • ✓ Código fácil de leer y mantener

Las llaves { } son elementos estructurales fundamentales que delimitan bloques de código y definen la organización jerárquica de un programa Java. Deben estar siempre balanceadas, y el código dentro de ellas debe estar correctamente indentado para facilitar la lectura y comprensión. Seguir las convenciones estándar de Java para el uso de llaves y la indentación desde el principio crea buenos hábitos que harán el código más profesional y mantenible.

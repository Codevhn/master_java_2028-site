4.10 Importación de clases y paquetes (import)

Java incluye miles de clases predefinidas organizadas en bibliotecas llamadas paquetes que proporcionan funcionalidad lista para usar: trabajar con fechas, leer archivos, realizar cálculos matemáticos, y mucho más. Para usar estas clases en nuestros programas, necesitamos importarlas mediante la palabra clave import. Comprender qué son los paquetes, cómo funcionan las importaciones, cuándo son necesarias, y cómo escribirlas correctamente es fundamental para aprovechar todo el poder de las bibliotecas estándar de Java y escribir programas más completos sin tener que reinventar funcionalidad básica.

¿Qué es un paquete?

Un paquete es una agrupación de clases relacionadas. Es como una carpeta que organiza clases por funcionalidad.

Ejemplos de paquetes de Java:

  • java.lang - Clases fundamentales (String, Math, System)
  • java.util - Utilidades (Scanner, ArrayList, Random)
  • java.time - Manejo de fechas y tiempos (LocalDate, LocalTime)
  • java.io - Entrada/salida de archivos (File, InputStream)

Analogía:

Los paquetes son como los departamentos de una biblioteca:

  • Departamento "java.util" - Libros de utilidades
  • Departamento "java.time" - Libros de fechas y tiempos
  • Cada libro es una clase que puede usar

¿Qué es la importación?

Importar es decirle a Java qué clases de otros paquetes queremos usar en nuestro programa.

Sintaxis básica:

import nombre.del.paquete.NombreClase;

Ejemplo:

import java.util.Scanner; // Importar la clase Scanner

Ahora podemos usar Scanner en nuestro programa.

El paquete java.lang (automático)

Hay UN paquete especial que Java importa automáticamente: java.lang

Este paquete contiene clases fundamentales que usamos constantemente:

  • String - Para trabajar con texto
  • System - Para entrada/salida (System.out.println)
  • Math - Para operaciones matemáticas

Por eso NUNCA necesitamos escribir:

import java.lang.String; // ✗ Innecesario
import java.lang.System; // ✗ Innecesario
import java.lang.Math;   // ✗ Innecesario

Java los importa automáticamente.

Ejemplo: Usar Math sin importar

public class CalculadoraRaiz {

    public static void main(String[] args) {

        double numero = 16.0;

        // Math está en java.lang (automático)
        double raizCuadrada = Math.sqrt(numero);

        System.out.println("Raíz cuadrada de " + numero + " = " + raizCuadrada);

    }

}

Salida:

Raíz cuadrada de 16.0 = 4.0

No necesitamos import java.lang.Math; porque está en java.lang

Ejemplo: Usar LocalDate (requiere import)

LocalDate está en el paquete java.time, NO en java.lang, entonces DEBEMOS importarla:

import java.time.LocalDate; // Importar LocalDate

public class FechaActual {

    public static void main(String[] args) {

        // Obtener fecha actual
        LocalDate hoy = LocalDate.now();

        System.out.println("Fecha actual: " + hoy);

    }

}

Salida (ejemplo):

Fecha actual: 2026-03-28

Sin el import, obtendríamos error de compilación.

¿Qué sucede sin el import?

Intento sin importar:

// Sin import
public class Prueba {

    public static void main(String[] args) {

        LocalDate fecha = LocalDate.now(); // ✗ Error

    }

}

Error de compilación:

error: cannot find symbol
        LocalDate fecha = LocalDate.now();
        ^
symbol:   class LocalDate
location: class Prueba

El compilador no sabe qué es LocalDate porque no está importada.

Ubicación de las sentencias import

Las sentencias import van al inicio del archivo, antes de la declaración de clase:

// 1. Imports (si hay)
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;

// 2. Declaración de clase
public class MiPrograma {

    public static void main(String[] args) {
        // Código
    }

}

Orden correcto:

1. imports
2. Declaración de clase
3. Contenido de clase

Ejemplo incorrecto (import después de clase):

public class MiPrograma { // ✗ Clase antes de import
    // ...
}

import java.time.LocalDate; // ✗ Error: import debe ir primero

Error de compilación.

Importar múltiples clases

Puede importar múltiples clases, cada una en su propia línea:

import java.time.LocalDate;
import java.time.LocalTime;
import java.time.format.DateTimeFormatter;

public class FechaHora {

    public static void main(String[] args) {

        LocalDate fecha = LocalDate.now();
        LocalTime hora = LocalTime.now();

        System.out.println("Fecha: " + fecha);
        System.out.println("Hora: " + hora);

    }

}

Importación con asterisco *

Puede importar todas las clases de un paquete con *:

import java.time.*; // Importa todas las clases de java.time

Ejemplo:

import java.time.*; // Importa LocalDate, LocalTime, etc.

public class Ejemplo {

    public static void main(String[] args) {

        LocalDate fecha = LocalDate.now();
        LocalTime hora = LocalTime.now();

        System.out.println("Fecha: " + fecha);
        System.out.println("Hora: " + hora);

    }

}

Diferencia entre importación específica y con *:

// Opción 1: Importar específicamente
import java.time.LocalDate;
import java.time.LocalTime;

// Opción 2: Importar todo el paquete
import java.time.*;

Ambas funcionan, pero la práctica recomendada es importar específicamente para claridad.

Ejemplo completo: Sistema de registro con fecha

import java.time.LocalDate;
import java.time.format.DateTimeFormatter;

public class RegistroVenta {

    public static void main(String[] args) {

        System.out.println("===== REGISTRO DE VENTA =====");

        // Obtener fecha actual
        LocalDate fechaVenta = LocalDate.now();

        // Formatear fecha a formato legible
        DateTimeFormatter formatoFecha = DateTimeFormatter.ofPattern("dd/MM/yyyy");
        String fechaFormateada = fechaVenta.format(formatoFecha);

        // Datos de la venta
        String nombreProducto = "Monitor LED 24 pulgadas";
        int cantidad = 3;
        double precioUnitario = 2500.00;
        double totalVenta = cantidad * precioUnitario;

        // Mostrar registro
        System.out.println("Fecha: " + fechaFormateada);
        System.out.println("Producto: " + nombreProducto);
        System.out.println("Cantidad: " + cantidad + " unidades");
        System.out.println("Precio unitario: Lps " + precioUnitario);
        System.out.println("-----------------------------");
        System.out.println("TOTAL: Lps " + totalVenta);

    }

}

Salida (ejemplo):

===== REGISTRO DE VENTA =====
Fecha: 28/03/2026
Producto: Monitor LED 24 pulgadas
Cantidad: 3 unidades
Precio unitario: Lps 2500.0
-----------------------------
TOTAL: Lps 7500.0

Imports necesarios:

  • LocalDate para obtener fecha actual
  • DateTimeFormatter para formatear la fecha

Clases comunes que requieren import

Para trabajar con fechas y tiempos:

import java.time.LocalDate;     // Fechas
import java.time.LocalTime;     // Horas
import java.time.LocalDateTime; // Fecha y hora

Para formatear fechas:

import java.time.format.DateTimeFormatter;

Para generar números aleatorios:

import java.util.Random;

Ejemplo:

import java.util.Random;

public class NumeroAleatorio {

    public static void main(String[] args) {

        Random generador = new Random();

        // Generar número entre 1 y 100
        int numeroAleatorio = generador.nextInt(100) + 1;

        System.out.println("Número aleatorio: " + numeroAleatorio);

    }

}

Ejemplo: Calculadora con funciones matemáticas

Usando Math (NO requiere import):

public class CalculadoraMatematica {

    public static void main(String[] args) {

        double numero = 25.0;

        // Math está en java.lang (automático)
        double raizCuadrada = Math.sqrt(numero);
        double potencia = Math.pow(numero, 2); // 25 al cuadrado
        double valorAbsoluto = Math.abs(-numero);

        System.out.println("===== CALCULADORA MATEMÁTICA =====");
        System.out.println("Número: " + numero);
        System.out.println("Raíz cuadrada: " + raizCuadrada);
        System.out.println("Al cuadrado: " + potencia);
        System.out.println("Valor absoluto de -" + numero + ": " + valorAbsoluto);

    }

}

Salida:

===== CALCULADORA MATEMÁTICA =====
Número: 25.0
Raíz cuadrada: 5.0
Al cuadrado: 625.0
Valor absoluto de -25.0: 25.0

No necesita imports porque Math está en java.lang

Errores comunes con imports

Error 1: Olvidar el import

// Sin import
public class Prueba {

    public static void main(String[] args) {

        LocalDate fecha = LocalDate.now(); // ✗ Error

    }

}

Solución: Agregar import java.time.LocalDate; al inicio

Error 2: Import después de la clase

public class Prueba {
    // ...
}

import java.time.LocalDate; // ✗ Lugar incorrecto

Solución: Mover import antes de la declaración de clase

Error 3: Punto y coma faltante en import

import java.time.LocalDate // ✗ Falta ;

Solución: Agregar ; al final

import java.time.LocalDate; // ✓ Correcto

Buenas prácticas con imports

1. Importar solo lo que se usa

// ✓ Solo lo necesario
import java.time.LocalDate;

// ✗ Importar de más
import java.time.*; // Si solo usas LocalDate

2. Organizar imports alfabéticamente

import java.time.LocalDate;
import java.time.LocalTime;
import java.time.format.DateTimeFormatter;

Los IDEs hacen esto automáticamente.

3. No importar clases de java.lang

// ✗ Innecesario
import java.lang.String;
import java.lang.Math;
import java.lang.System;

Ya están disponibles automáticamente.

4. Agrupar imports del mismo paquete

// Imports de java.time juntos
import java.time.LocalDate;
import java.time.LocalTime;

// Imports de java.util juntos
import java.util.Random;

Ejemplo final: Sistema de cálculo de días laborables

import java.time.LocalDate;
import java.time.DayOfWeek;

public class CalculadoraDiasLaborables {

    public static void main(String[] args) {

        System.out.println("===== CÁLCULO DE DÍAS LABORABLES =====");

        // Fecha de hoy
        LocalDate hoy = LocalDate.now();

        // Obtener día de la semana
        DayOfWeek diaSemana = hoy.getDayOfWeek();

        // Datos del mes
        int diaDelMes = hoy.getDayOfMonth();
        int mes = hoy.getMonthValue();
        int anio = hoy.getYear();

        // Mostrar información
        System.out.println("Fecha actual: " + diaDelMes + "/" + mes + "/" + anio);
        System.out.println("Día de la semana: " + diaSemana);

        // Determinar si es día laborable
        boolean esDiaLaborable = (diaSemana != DayOfWeek.SATURDAY &&
                                   diaSemana != DayOfWeek.SUNDAY);

        System.out.println("-------------------------------------");

        if (esDiaLaborable) {
            System.out.println("Es día laborable");
        } else {
            System.out.println("Es fin de semana");
        }

    }

}

Imports necesarios:

  • LocalDate - Para trabajar con fechas
  • DayOfWeek - Para obtener día de la semana

Salida (ejemplo un viernes):

===== CÁLCULO DE DÍAS LABORABLES =====
Fecha actual: 28/3/2026
Día de la semana: FRIDAY
-------------------------------------
Es día laborable

Resumen de conceptos clave

¿Qué es un paquete?

  • Agrupación de clases relacionadas

¿Qué es import?

  • Instrucción para usar clases de otros paquetes

¿Dónde van los imports?

  • Al inicio del archivo, antes de la declaración de clase

¿Cuándo NO necesito import?

  • Para clases en java.lang (String, Math, System)

Sintaxis:

import nombre.paquete.NombreClase;

Las sentencias import nos permiten acceder a la vasta biblioteca de clases que Java proporciona, evitando tener que escribir todo desde cero. Comprender qué clases necesitan importación, cómo escribir las sentencias import correctamente, y dónde ubicarlas en el archivo nos permite aprovechar toda la funcionalidad que Java ofrece para crear programas más completos y potentes.

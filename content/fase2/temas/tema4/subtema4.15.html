4.15: Errores estructurales comunes en programas Java

Los errores estructurales son aquellos que violan las reglas fundamentales de organización y sintaxis del lenguaje Java. A diferencia de errores lógicos que producen resultados incorrectos pero permiten que el programa compile y ejecute, los errores estructurales impiden que el código compile o ejecute correctamente. Estos errores son particularmente comunes en programadores principiantes que aún están internalizando las reglas estrictas de Java sobre cómo debe estructurarse el código.

La buena noticia sobre errores estructurales es que el compilador los detecta automáticamente y proporciona mensajes indicando exactamente qué está mal y dónde. La mala noticia es que los mensajes del compilador pueden ser crípticos para principiantes que aún no han desarrollado la habilidad de traducir la terminología técnica del compilador a conceptos comprensibles. Este subtema cataloga los errores estructurales más comunes, explica por qué ocurren, muestra cómo reconocerlos por sus mensajes de error, y proporciona soluciones claras.

Comprender estos errores comunes acelera dramáticamente el proceso de aprendizaje porque permite reconocer patrones. Cuando se ve un mensaje de error particular por quinta o décima vez, el cerebro automáticamente sabe qué está mal y cómo corregirlo sin necesidad de analizar detenidamente el mensaje. Esta familiaridad con errores comunes es una de las diferencias clave entre principiantes que luchan con cada error y programadores con experiencia que corrigen errores rápidamente casi por reflejo.

Error estructural 1: Olvidar declarar la clase

Java requiere que todo código ejecutable esté dentro de una clase. No puede haber código "suelto" en un archivo .java.

  // Error: código sin clase contenedora
public static void main(String[] args) {
    System.out.println("Hola");
}

Mensaje de error:

error: class, interface, or enum expected

El compilador espera encontrar una declaración de clase, interfaz, o enumeración, pero en su lugar encuentra directamente un método.

Solución:

  public class MiPrograma {
    public static void main(String[] args) {
        System.out.println("Hola");
    }
}

Todo el código debe estar encerrado dentro de la declaración de clase con sus llaves correspondientes.

Error estructural 2: Nombre de clase no coincide con nombre de archivo

La clase pública debe tener exactamente el mismo nombre que el archivo que la contiene.

Archivo: Programa.java

  public class Calculadora {
    public static void main(String[] args) {
        System.out.println("Calculando...");
    }
}

Mensaje de error:

Programa.java:1: error: class Calculadora is public, should be declared in a file named Calculadora.java
public class Calculadora {
       ^

El compilador detecta que el nombre de la clase (Calculadora) no coincide con el nombre del archivo (Programa.java).

Soluciones:

Opción 1: Renombrar el archivo para que coincida con la clase:

  mv Programa.java Calculadora.java

Opción 2: Cambiar el nombre de la clase para que coincida con el archivo:

  public class Programa {
    public static void main(String[] args) {
        System.out.println("Calculando...");
    }
}

Error estructural 3: Método main con firma incorrecta

El método main debe tener exactamente la firma public static void main(String[] args). Cualquier variación hace que la JVM no lo reconozca como punto de entrada.

Error: Falta static

  public class Ejemplo {
    public void main(String[] args) { // Falta static
        System.out.println("Hola");
    }
}

Compilación: Exitosa (no hay error de compilación)

Error en ejecución:

Error: Main method is not static in class Ejemplo, please define the main method as:
   public static void main(String[] args)

Solución:

  public static void main(String[] args) { // Correcto

Error: Falta public

  public class Ejemplo {
    static void main(String[] args) { // Falta public
        System.out.println("Hola");
    }
}

Mensaje de error en ejecución:

Error: Main method not found in class Ejemplo

Solución:

  public static void main(String[] args) { // Correcto

Error: Tipo de retorno incorrecto

  public class Ejemplo {
    public static int main(String[] args) { // int en vez de void
        System.out.println("Hola");
        return 0;
    }
}

Mensaje de error en ejecución:

Error: Main method must return a value of type void in class Ejemplo

Solución:

  public static void main(String[] args) { // void es correcto

Error: Nombre incorrecto (mayúscula)

  public class Ejemplo {
    public static void Main(String[] args) { // Main con mayúscula
        System.out.println("Hola");
    }
}

Mensaje de error en ejecución:

Error: Main method not found in class Ejemplo

La JVM busca main con minúscula inicial, no Main con mayúscula.

Solución:

  public static void main(String[] args) { // Minúscula

Error estructural 4: Llaves desbalanceadas

Cada llave de apertura { debe tener su correspondiente llave de cierre }.

Falta llave de cierre del método:

  public class ErrorLlaves {
    public static void main(String[] args) {
        double precio = 100.0;
        System.out.println(precio);
        // Falta cerrar main
}

Mensaje de error:

ErrorLlaves.java:6: error: reached end of file while parsing
}
^

El compilador llegó al final del archivo esperando encontrar más código (la llave de cierre del método main).

Solución:

  public class ErrorLlaves {
    public static void main(String[] args) {
        double precio = 100.0;
        System.out.println(precio);
    } // Cierra main
} // Cierra clase

Llave extra:

  public class ErrorLlaveExtra {
    public static void main(String[] args) {
        System.out.println("Hola");
    }
}
} // Llave extra

Mensaje de error:

ErrorLlaveExtra.java:6: error: class, interface, or enum expected
}
^

El compilador encontró una llave de cierre que no corresponde a ninguna apertura.

Solución: Eliminar la llave extra.

Error estructural 5: Punto y coma faltante

Cada sentencia ejecutable debe terminar con punto y coma.

  public class ErrorPuntoYComa {
    public static void main(String[] args) {
        double precio = 500.00 // Falta ;
        double descuento = 75.00;
        System.out.println(precio);
    }
}

Mensaje de error:

ErrorPuntoYComa.java:3: error: ';' expected
        double precio = 500.00
                             ^

Solución:

  double precio = 500.00; // Agregado ;

Punto y coma después de llaves (error sutil):

  public class ErrorPuntoYComaDespuesLlave {
    public static void main(String[] args) {
        System.out.println("Hola");
    }; // Error: ; después de }
}

Esto técnicamente compila pero el ; después de } se interpreta como sentencia vacía. No causa error pero es incorrecto estilísticamente.

Correcto:

  public static void main(String[] args) {
    System.out.println("Hola");
} // Sin ;

Error estructural 6: Variables no declaradas o usadas antes de declaración

Una variable debe ser declarada antes de poder usarse.

  public class ErrorVariableNoDeclarada {
    public static void main(String[] args) {
        precio = 100.0; // precio nunca fue declarada
        System.out.println(precio);
    }
}

Mensaje de error:

ErrorVariableNoDeclarada.java:3: error: cannot find symbol
        precio = 100.0;
        ^
symbol:   variable precio
location: class ErrorVariableNoDeclarada

Solución:

  double precio = 100.0; // Declarar primero

Usar variable antes de declararla:

  public class ErrorOrdenDeclaracion {
    public static void main(String[] args) {
        total = precio + impuesto; // precio e impuesto no existen aún
        double precio = 100.0;
        double impuesto = 15.0;
    }
}

Solución: Declarar variables antes de usarlas.

  double precio = 100.0;
double impuesto = 15.0;
double total = precio + impuesto; // Ahora precio e impuesto existen

Error estructural 7: Tipos de datos incompatibles

Asignar un valor de un tipo a una variable de otro tipo incompatible.

  public class ErrorTipoIncompatible {
    public static void main(String[] args) {
        int cantidad = 25.5; // Asignar double a int
    }
}

Mensaje de error:

ErrorTipoIncompatible.java:3: error: incompatible types: possible lossy conversion from double to int
        int cantidad = 25.5;
                       ^

El compilador detecta que 25.5 es tipo double pero se intenta asignar a tipo int, lo que perdería la parte decimal.

Solución: Usar tipo apropiado.

  double cantidad = 25.5; // Correcto

O si realmente se necesita entero:

  int cantidad = 25; // Correcto

Asignar texto a número:

  public class ErrorTextoANumero {
    public static void main(String[] args) {
        double precio = "100.00"; // String a double
    }
}

Mensaje de error:

ErrorTextoANumero.java:3: error: incompatible types: String cannot be converted to double
        double precio = "100.00";
                        ^

Solución:

  double precio = 100.00; // Sin comillas

Error estructural 8: Imports incorrectos o faltantes

Usar una clase que requiere importación sin importarla.

  public class ErrorImportFaltante {
    public static void main(String[] args) {
        LocalDate fecha = LocalDate.now(); // LocalDate no importado
    }
}

Mensaje de error:

ErrorImportFaltante.java:3: error: cannot find symbol
        LocalDate fecha = LocalDate.now();
        ^
symbol:   class LocalDate
location: class ErrorImportFaltante

Solución: Agregar import.

  import java.time.LocalDate;

public class ErrorImportFaltante {
    public static void main(String[] args) {
        LocalDate fecha = LocalDate.now(); // Correcto
    }
}

Import en lugar incorrecto (después de clase):

  public class ErrorImportUbicacion {
    public static void main(String[] args) {
        // ...
    }
}

import java.time.LocalDate; // Error: import después de clase

Mensaje de error:

error: class, interface, or enum expected
import java.time.LocalDate;
^

Solución: Mover imports antes de la declaración de clase.

  import java.time.LocalDate; // Correcto: antes de clase

public class ErrorImportUbicacion {
    // ...
}

Error estructural 9: Caracteres especiales inválidos

Usar caracteres que Java no reconoce en el código. Este error es sutil porque visualmente puede parecer correcto, pero si se copió código de un documento de Word o página web, las comillas pueden ser caracteres Unicode especiales en lugar de las comillas ASCII que Java espera.

  public class ErrorCaracteres {
    public static void main(String[] args) {
        double precio = 100.00;
        System.out.println("Precio: " + precio); // Comillas especiales " " en vez de " "
    }
}

Mensaje de error:

ErrorCaracteres.java:4: error: illegal character: '\u201c'
        System.out.println("Precio: " + precio);
                           ^

Solución: Reescribir las comillas usando el teclado directamente en el editor de código, no copiar de documentos formateados.

Error estructural 10: Código inalcanzable

Código que nunca puede ejecutarse porque está después de una sentencia que siempre termina la ejecución.

  public class ErrorCodigoInalcanzable {
    public static void main(String[] args) {
        System.out.println("Inicio");
        return; // Termina main inmediatamente
        System.out.println("Este código nunca se ejecuta"); // Inalcanzable
    }
}

Mensaje de error:

ErrorCodigoInalcanzable.java:5: error: unreachable statement
        System.out.println("Este código nunca se ejecuta");
        ^

El compilador detecta que esta línea nunca podrá ejecutarse porque el return anterior siempre termina el método.

Solución: Eliminar el código inalcanzable o reestructurar la lógica.

Error estructural 11: Mayúsculas y minúsculas incorrectas

Java distingue entre mayúsculas y minúsculas. Precio y precio son variables completamente diferentes.

  public class ErrorMayusculas {
    public static void main(String[] args) {
        double Precio = 100.0; // Con mayúscula
        System.out.println(precio); // Con minúscula
    }
}

Mensaje de error:

ErrorMayusculas.java:4: error: cannot find symbol
        System.out.println(precio);
                           ^
symbol:   variable precio
location: class ErrorMayusculas

El compilador busca precio (minúscula) pero solo existe Precio (mayúscula).

Solución: Usar nombre consistente.

  double precio = 100.0; // Minúscula (convención correcta)
System.out.println(precio); // Minúscula

Estrategias para prevenir errores estructurales

Usar un IDE. Los IDEs modernos como IntelliJ IDEA, Eclipse, o VS Code detectan muchos errores estructurales mientras se escribe el código, antes de compilar. Subrayados rojos indican errores, amarillos indican advertencias.

Compilar frecuentemente. No esperar a terminar todo el programa para compilar. Compilar después de agregar cada pocas líneas permite detectar errores cuando son más fáciles de localizar.

Leer mensajes de error completos. No solo mirar el número de línea. Leer la descripción completa del error para entender qué está mal.

Mantener indentación correcta. La indentación apropiada hace que llaves desbalanceadas sean visualmente obvias.

Usar nombres de variables descriptivos. Nombres largos y descriptivos reducen errores de tipeo y confusión con mayúsculas/minúsculas.

Verificar tipos de datos. Antes de asignar un valor a una variable, verificar mentalmente que los tipos coinciden.

Los errores estructurales son frustrantes cuando se está aprendiendo Java porque impiden que el código compile o ejecute. Sin embargo, estos errores son en realidad aliados porque el compilador los detecta automáticamente antes de que el código defectuoso cause problemas más serios durante la ejecución. Con práctica, los mensajes de error del compilador se vuelven cada vez más fáciles de interpretar, y muchos de estos errores se evitan completamente al desarrollar buenos hábitos de escritura de código desde el principio.

4.2 Concepto de archivo fuente (.java)

El archivo con extensión .java es donde escribimos nuestros programas Java. Es un archivo de texto simple y común que cualquier editor puede abrir, pero que sigue reglas muy específicas sobre su nombre, contenido y organización. Comprender qué es exactamente un archivo .java, cómo debe llamarse, dónde debe ubicarse, y qué reglas debe seguir es fundamental para evitar errores de compilación y escribir programas que funcionen correctamente. Este subtema explora el concepto de archivo fuente Java en profundidad, estableciendo las bases para comprender cómo organizar correctamente nuestro código.

¿Qué es un archivo .java?

Un archivo .java es:

1. Un archivo de texto plano

  • No es un documento de Word, PDF, o archivo binario
  • Puede abrirse con cualquier editor de texto (Notepad, VS Code, IntelliJ, etc.)
  • No contiene formato especial (negrita, colores, fuentes)
  • El código que escribimos es texto legible por humanos

2. Con extensión .java obligatoria

  • El nombre del archivo DEBE terminar en .java
  • MiPrograma.java ✓ Correcto
  • MiPrograma.txt ✗ No funcionará
  • MiPrograma ✗ Sin extensión, no válido

3. Contiene código Java

  • Código escrito siguiendo la sintaxis del lenguaje Java
  • Organizado en clases (todo debe estar dentro de una clase)
  • Sigue reglas estrictas de nombres y estructura

Cómo crear un archivo .java

Opción 1: Usar un editor de texto simple

Windows:

1. Abrir Notepad (Bloc de notas)
2. Escribir el código
3. Guardar como "MiPrograma.java"
IMPORTANTE: En "Tipo", seleccionar "Todos los archivos (*.*)"
Si no, se guardará como "MiPrograma.java.txt"

macOS/Linux:

# Usar nano, vim, o cualquier editor
nano MiPrograma.java

# Escribir código, guardar (Ctrl+O, Enter, Ctrl+X)

Opción 2: Usar un IDE (recomendado)

Los IDEs (IntelliJ IDEA, Eclipse, VS Code) crean el archivo automáticamente con la estructura correcta y evitan errores comunes de nombres.

Regla fundamental: El nombre del archivo debe coincidir con el nombre de la clase

Esta es la regla más importante y la fuente más común de errores para principiantes.

Ejemplo correcto:

// Archivo: Calculadora.java

class Calculadora {
    public static void main(String[] args) {
        System.out.println("Calculadora funcionando");
    }
}

Correspondencia:

  • Nombre de clase: Calculadora
  • Nombre de archivo: Calculadora.java
  • ✓ Coinciden exactamente

Ejemplo incorrecto:

// Archivo: Programa.java (nombre diferente)

class Calculadora {
    public static void main(String[] args) {
        System.out.println("No funcionará");
    }
}

Intento de compilar:

javac Programa.java

Error:

Programa.java:3: error: class Calculadora is public, should be declared in a file named Calculadora.java
class Calculadora {
^

El compilador dice claramente: la clase Calculadora debe estar en un archivo llamado Calculadora.java.

Sensibilidad a mayúsculas y minúsculas

Java distingue entre mayúsculas y minúsculas. Calculadora y calculadora son DIFERENTES.

Ejemplo de problema:

// Archivo: calculadora.java (todo minúsculas)

class Calculadora { // Nombre con mayúscula
    public static void main(String[] args) {
        System.out.println("Hola");
    }
}

En Windows:

  • Puede funcionar (Windows no distingue mayúsculas/minúsculas en nombres de archivo)
  • Pero NO es recomendado, causará problemas en otros sistemas

En macOS/Linux:

javac calculadora.java
# Error: nombres no coinciden

Solución: Siempre coincidir exactamente:

// Archivo: Calculadora.java
class Calculadora {
    // ✓ Correcto
}

Estructura básica de un archivo .java

Un archivo .java típico tiene esta estructura simple:

// 1. Comentarios (opcional)
// Descripción de lo que hace el programa

// 2. Declaración de la clase (obligatorio)
class NombreClase {

    // 3. Método main (obligatorio en programa ejecutable)
    public static void main(String[] args) {

        // 4. Código del programa (instrucciones)
        System.out.println("Hola mundo");
    }
}

Ejemplo completo de archivo .java

Archivo: CalculadoraSalario.java

// Calculadora de salario semanal
// Calcula el salario basado en horas trabajadas y tarifa por hora

class CalculadoraSalario {
    public static void main(String[] args) {
        // Datos del empleado
        double horasTrabajadas = 40.0;
        double tarifaPorHora = 125.00;

        // Calcular salario semanal
        double salarioSemanal = horasTrabajadas * tarifaPorHora;

        // Mostrar resultado
        System.out.println("=== Calculadora de Salario ===");
        System.out.println("Horas trabajadas: " + horasTrabajadas);
        System.out.println("Tarifa por hora: Lps " + tarifaPorHora);
        System.out.println("Salario semanal: Lps " + salarioSemanal);
    }
}

Guardar como: CalculadoraSalario.java

Compilar:

javac CalculadoraSalario.java

Resultado: Se crea CalculadoraSalario.class

Ejecutar:

java CalculadoraSalario

Salida:

=== Calculadora de Salario ===
Horas trabajadas: 40.0
Tarifa por hora: Lps 125.0
Salario semanal: Lps 5000.0

Relación entre archivo .java y archivo .class

Proceso:

1. Escribir código
↓
CalculadoraSalario.java (código fuente, legible)

2. Compilar con javac
↓
javac CalculadoraSalario.java
↓
CalculadoraSalario.class (bytecode, no legible)

3. Ejecutar con java
↓
java CalculadoraSalario
↓
Programa ejecutándose

Diferencias clave:

Archivo .java Archivo .class
Código fuente Bytecode
Legible por humanos No legible por humanos
Editamos este archivo Generado automáticamente
Se versiona en Git NO se versiona en Git
Texto plano Binario

¿Dónde debe ubicarse el archivo .java?

Para programas simples (por ahora):

mi-proyecto/
├── CalculadoraSalario.java
├── CalculadoraDescuento.java
└── ProgramaInventario.java

Todos los archivos .java pueden estar en el mismo directorio.

Compilar desde ese directorio:

cd mi-proyecto
javac CalculadoraSalario.java
java CalculadoraSalario

Convenciones de nombres para archivos .java

Reglas obligatorias:

  1. Debe terminar en .java
  2. Debe coincidir con el nombre de la clase
  3. Distingue mayúsculas/minúsculas

Convenciones recomendadas (estilo Java):

✓ Correcto (PascalCase):

CalculadoraImpuestos.java
SistemaInventario.java
ProcesadorVentas.java
ReporteFacturas.java

Cada palabra empieza con mayúscula, sin espacios ni guiones.

✗ No recomendado (funcionan pero no son estilo Java):

calculadora_impuestos.java // Estilo snake_case (Python/C)
calculadoraimpuestos.java  // Todo minúsculas
CALCULADORAIMPUESTOS.java  // Todo mayúsculas
calculadora-impuestos.java // Kebab-case
calculadora impuestos.java // Espacios (inválido en muchos sistemas)

Ejemplo: Sistema de cálculo de inventario

Archivo: CalculadoraInventario.java

// Sistema de cálculo de valor de inventario
// Calcula el valor total basado en cantidades y precios

class CalculadoraInventario {
    public static void main(String[] args) {
        // Producto 1: Laptops
        int cantidadLaptops = 15;
        double precioLaptop = 12000.00;
        double valorLaptops = cantidadLaptops * precioLaptop;

        // Producto 2: Teclados
        int cantidadTeclados = 50;
        double precioTeclado = 350.00;
        double valorTeclados = cantidadTeclados * precioTeclado;

        // Producto 3: Monitores
        int cantidadMonitores = 25;
        double precioMonitor = 2500.00;
        double valorMonitores = cantidadMonitores * precioMonitor;

        // Calcular valor total del inventario
        double valorTotalInventario = valorLaptops + valorTeclados + valorMonitores;

        // Mostrar reporte
        System.out.println("========== INVENTARIO ==========");
        System.out.println("Laptops: " + cantidadLaptops + " x Lps " + precioLaptop + " = Lps " + valorLaptops);
        System.out.println("Teclados: " + cantidadTeclados + " x Lps " + precioTeclado + " = Lps " + valorTeclados);
        System.out.println("Monitores: " + cantidadMonitores + " x Lps " + precioMonitor + " = Lps " + valorMonitores);
        System.out.println("================================");
        System.out.println("Valor Total: Lps " + valorTotalInventario);
    }
}

Salida:

========== INVENTARIO ==========
Laptops: 15 x Lps 12000.0 = Lps 180000.0
Teclados: 50 x Lps 350.0 = Lps 17500.0
Monitores: 25 x Lps 2500.0 = Lps 62500.0
================================
Valor Total: Lps 260000.0

Errores comunes con archivos .java

Error 1: Extensión incorrecta

Guardado como: MiPrograma.txt

Intento de compilar:

javac MiPrograma.txt
# Error: file not found

Solución: Cambiar nombre a MiPrograma.java

Error 2: Nombre no coincide con clase

// Archivo: Programa.java

class CalculadoraVentas {
    public static void main(String[] args) {
        System.out.println("Hola");
    }
}

Error de compilación: Nombres no coinciden

Solución: Cambiar nombre de archivo a CalculadoraVentas.java

Error 3: Mayúsculas/minúsculas incorrectas

// Archivo: calculadora.java (minúsculas)

class Calculadora { // Mayúscula inicial
    // ...
}

En Linux/macOS: Error de compilación

Solución: Renombrar archivo a Calculadora.java

Error 4: Doble extensión (Windows)

Al guardar en Notepad, si no selecciona "Todos los archivos", puede quedar:

MiPrograma.java.txt // ✗ Doble extensión

Solución:

  1. En Notepad, al guardar, en "Tipo" seleccionar "Todos los archivos (*.*)"
  2. O renombrar el archivo en el explorador mostrando extensiones

Buenas prácticas con archivos .java

1. Un archivo por programa

Para programas simples, mantenga cada programa en su propio archivo:

proyecto/
├── CalculadoraDescuento.java
├── CalculadoraSalario.java
└── ReporteInventario.java

No mezcle múltiples programas en un archivo.

2. Nombres descriptivos

// ✓ Descriptivo
CalculadoraImpuestoVentas.java
SistemaRegistroEmpleados.java

// ✗ Genérico
Programa1.java
Calc.java
Test.java

3. Mantener archivos organizados

mis-programas/
├── calculadoras/
│   ├── CalculadoraDescuento.java
│   └── CalculadoraImpuesto.java
└── reportes/
    ├── ReporteVentas.java
    └── ReporteInventario.java

4. Evitar caracteres especiales en nombres

// ✓ Solo caracteres simples
CalculadoraBasica.java

// ✗ Caracteres especiales
Calculadora-Básica.java // Guión y tilde
Calculadora#1.java      // Símbolo #

Ejemplo: Calculadora de préstamo simple

Archivo: CalculadoraPrestamo.java

// Calculadora de cuota mensual de préstamo
// Cálculo simple sin intereses

class CalculadoraPrestamo {
    public static void main(String[] args) {
        // Datos del préstamo
        double montoPrestamo = 50000.00;
        int plazomeses = 12;

        // Calcular cuota mensual (sin interés, solo ejemplo)
        double cuotaMensual = montoPrestamo / plazomeses;

        // Mostrar información
        System.out.println("===== PRÉSTAMO =====");
        System.out.println("Monto prestado: Lps " + montoPrestamo);
        System.out.println("Plazo: " + plazomeses + " meses");
        System.out.println("Cuota mensual: Lps " + cuotaMensual);
        System.out.println("====================");
    }
}

Compilar y ejecutar:

javac CalculadoraPrestamo.java
java CalculadoraPrestamo

Salida:

===== PRÉSTAMO =====
Monto prestado: Lps 50000.0
Plazo: 12 meses
Cuota mensual: Lps 4166.666666666667
====================

Verificar archivos creados

Después de compilar, puede verificar qué archivos se crearon:

Windows:

dir

macOS/Linux:

ls -l

Debería ver:

CalculadoraPrestamo.java  (archivo fuente)
CalculadoraPrestamo.class (archivo compilado)

El archivo .java es la base de todo desarrollo en Java. Es donde escribimos nuestro código, y debe seguir reglas estrictas de nomenclatura y organización para que el compilador pueda procesarlo correctamente. Comprender estas reglas desde el inicio evita errores frustrantes y establece buenos hábitos de programación que facilitarán el trabajo a medida que los programas se vuelvan más complejos.

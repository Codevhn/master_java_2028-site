4.14: Compilación de la estructura del programa

La compilación es el proceso que transforma el código fuente Java que escribimos en archivos .java en bytecode almacenado en archivos .class que la JVM puede ejecutar. Este proceso no es una simple traducción sino una transformación compleja que involucra análisis sintáctico, verificación de tipos, resolución de referencias, y generación de código intermedio. Comprender qué sucede durante la compilación, qué verifica el compilador, qué errores puede detectar, y cómo interpretar los mensajes del compilador es esencial para desarrollar código Java correctamente y resolver problemas cuando surgen.

El compilador de Java, javac, es más que un traductor de código. Es un verificador estricto que rechaza código que viola las reglas del lenguaje Java, incluso si ese código podría tener algún significado razonable. Esta estrictez puede parecer frustrante cuando el compilador rechaza código que "debería funcionar", pero en realidad esta verificación rigurosa previene innumerables errores que de otro modo solo se descubrirían durante la ejecución, cuando son más difíciles y costosos de corregir.

Cuando ejecutamos javac, iniciamos un proceso de múltiples etapas que examina cada aspecto de nuestro código. El compilador verifica la sintaxis, los tipos de datos, la accesibilidad de variables y métodos, la existencia de clases referenciadas, y docenas de otras reglas. Solo si el código pasa todas estas verificaciones, el compilador genera el archivo .class. Si hay aunque sea un error, la compilación falla y no se genera ningún archivo .class, garantizando que código con errores conocidos nunca llegue a ejecutarse.

El comando javac

La compilación se invoca mediante el comando javac seguido del nombre del archivo fuente:

  javac CalculadoraDescuento.java

Este comando simple desencadena el compilador Java. Si la compilación es exitosa, javac no produce ninguna salida visible, solo crea silenciosamente el archivo CalculadoraDescuento.class en el mismo directorio. La ausencia de mensajes es señal de éxito. Si hay errores, javac muestra mensajes detallados describiendo cada problema encontrado.

Consideremos un programa simple:

  public class CalculadoraDescuento {
    public static void main(String[] args) {
        double precio = 500.00;
        double descuento = precio * 0.15;
        double precioFinal = precio - descuento;
        System.out.println("Precio final: Lps " + precioFinal);
    }
}

Compilación exitosa:

  javac CalculadoraDescuento.java

Resultado: Silencio. Ningún mensaje. El compilador crea CalculadoraDescuento.class exitosamente.

Verificar que el archivo .class fue creado:

  ls -l
# Muestra:
# CalculadoraDescuento.java
# CalculadoraDescuento.class

El archivo .class es bytecode binario, no es legible por humanos. Si intentamos abrirlo con un editor de texto, veremos caracteres sin sentido porque está en formato binario diseñado para la JVM, no para lectura humana.

Fases del proceso de compilación

Aunque desde nuestra perspectiva la compilación parece instantánea, internamente el compilador ejecuta múltiples fases:

Fase 1: Análisis léxico

El compilador lee el archivo .java como secuencia de caracteres y lo divide en tokens (palabras clave, identificadores, operadores, literales, símbolos de puntuación). Por ejemplo, la línea double precio = 500.00; se divide en los tokens: double, precio, =, 500.00, ;.

Si el código contiene caracteres inválidos o combinaciones de caracteres que no forman tokens válidos, el análisis léxico falla y reporta error.

Fase 2: Análisis sintáctico

El compilador verifica que los tokens forman estructuras sintácticas válidas según la gramática de Java. Por ejemplo, double precio = 500.00; es sintácticamente válido (declaración con inicialización), pero double = precio 500.00; es sintácticamente inválido porque los tokens están en orden incorrecto.

Esta fase construye un árbol de sintaxis abstracta que representa la estructura del programa.

Fase 3: Análisis semántico

El compilador verifica el significado del código. Aquí se detectan errores como:

  • Usar una variable antes de declararla
  • Asignar un valor de tipo incorrecto a una variable
  • Llamar métodos que no existen
  • Referenciar clases que no están disponibles

Esta es la fase donde la mayoría de errores de programación se detectan.

Fase 4: Generación de bytecode

Si todas las fases anteriores completan sin errores, el compilador genera las instrucciones de bytecode correspondientes al código fuente y las escribe en el archivo .class.

Errores de compilación comunes

Error: Olvidar punto y coma

  public class ErrorPuntoYComa {
    public static void main(String[] args) {
        double precio = 100.0 // Falta ;
        System.out.println(precio);
    }
}

Mensaje del compilador:

ErrorPuntoYComa.java:3: error: ';' expected
        double precio = 100.0
                             ^
1 error

El compilador indica exactamente dónde esperaba encontrar el punto y coma. El símbolo ^ apunta al lugar después del último token correcto donde debería estar el ;.

Corrección: Agregar ; al final de la línea.

  double precio = 100.0; // Correcto

Error: Tipo de dato incorrecto

  public class ErrorTipo {
    public static void main(String[] args) {
        int cantidad = 25.5; // Intentar asignar decimal a int
    }
}

Mensaje del compilador:

ErrorTipo.java:3: error: incompatible types: possible lossy conversion from double to int
        int cantidad = 25.5;
                       ^
1 error

El compilador explica que 25.5 es de tipo double (decimal) pero se intenta asignar a una variable de tipo int (entero), y esto podría perder información (la parte decimal).

Corrección: Usar el tipo correcto.

  double cantidad = 25.5; // Correcto

O si realmente se necesita un entero, usar valor entero:

  int cantidad = 25; // Correcto

Error: Variable no declarada

  public class ErrorVariableNoDeclarada {
    public static void main(String[] args) {
        precio = 100.0; // precio no fue declarada
        System.out.println(precio);
    }
}

Mensaje del compilador:

ErrorVariableNoDeclarada.java:3: error: cannot find symbol
        precio = 100.0;
        ^
symbol:   variable precio
location: class ErrorVariableNoDeclarada
1 error

El compilador no puede "encontrar el símbolo" precio porque nunca fue declarado.

Corrección: Declarar la variable.

  double precio = 100.0; // Correcto

Error: Llaves desbalanceadas

  public class ErrorLlaves {
    public static void main(String[] args) {
        double precio = 100.0;
        System.out.println(precio);
    }
    // Falta cerrar llave de clase

Mensaje del compilador:

ErrorLlaves.java:6: error: reached end of file while parsing
}
^
1 error

El compilador llegó al final del archivo mientras esperaba encontrar más código (específicamente, la llave de cierre de la clase).

Corrección: Agregar la llave faltante.

  public class ErrorLlaves {
    public static void main(String[] args) {
        double precio = 100.0;
        System.out.println(precio);
    }
} // Correcto

Error: Nombre de clase no coincide con nombre de archivo

Archivo: Programa.java

  public class Calculadora { // Nombre no coincide con archivo
    public static void main(String[] args) {
        System.out.println("Hola");
    }
}

Mensaje del compilador:

Programa.java:1: error: class Calculadora is public, should be declared in a file named Calculadora.java
public class Calculadora {
       ^
1 error

Corrección: Renombrar archivo a Calculadora.java o cambiar nombre de clase a Programa.

Interpretar mensajes de error del compilador

Los mensajes de error de javac siguen un formato consistente que facilita su interpretación:

NombreArchivo.java:NumeroLinea: error: Descripción del error
Línea de código donde ocurrió el error
^

Componentes del mensaje:

  • NombreArchivo.java — Archivo que contiene el error
  • NumeroLinea — Número de línea donde el compilador detectó el error
  • error: — Indica que es un error (no advertencia)
  • Descripción — Explica qué está mal
  • Línea de código — Muestra la línea problemática
  • ^ — Apunta al lugar específico del error

Ejemplo de interpretación:

CalculadoraDescuento.java:5: error: ';' expected
        double descuento = precio * 0.15
                                       ^

Interpretación:

  • Archivo: CalculadoraDescuento.java
  • Línea: 5
  • Problema: Falta punto y coma
  • Ubicación exacta: Después de 0.15

Acción: Ir a línea 5, agregar ; al final.

Múltiples errores en una compilación

Si el código tiene múltiples errores, el compilador intenta detectar tantos como sea posible en una sola pasada y reporta todos:

  public class MultiplesErrores {
    public static void main(String[] args) {
        double precio = 500.00
        int cantidad = 5.5;
        double total = precio * cantidad
        System.out.println(total)
    }
}

Mensajes del compilador:

MultiplesErrores.java:3: error: ';' expected
        double precio = 500.00
                             ^
MultiplesErrores.java:4: error: incompatible types: possible lossy conversion from double to int
        int cantidad = 5.5;
                       ^
MultiplesErrores.java:5: error: ';' expected
        double total = precio * cantidad
                                        ^
MultiplesErrores.java:6: error: ';' expected
        System.out.println(total)
                                 ^
4 errors

El compilador encontró 4 errores y los reportó todos. La estrategia más efectiva es corregir los errores uno por uno, comenzando desde el primero, porque errores tempranos frecuentemente causan errores en cascada que desaparecen al corregir el error original.

Corrección:

  public class MultiplesErrores {
    public static void main(String[] args) {
        double precio = 500.00;   // Agregado ;
        double cantidad = 5.5;    // Cambiado int a double
        double total = precio * cantidad;  // Agregado ;
        System.out.println(total);         // Agregado ;
    }
}

Compilar múltiples archivos

Si un proyecto tiene múltiples archivos .java, se pueden compilar todos juntos:

  javac *.java

Esto compila todos los archivos .java en el directorio actual. El compilador determina automáticamente las dependencias entre archivos y los compila en el orden correcto.

También se pueden especificar múltiples archivos explícitamente:

  javac Calculadora.java Reporte.java Sistema.java

Si algún archivo tiene errores, la compilación completa falla y se muestran errores de todos los archivos problemáticos.

Opciones útiles de javac

Especificar directorio de salida:

  javac -d bin CalculadoraDescuento.java

Esto compila CalculadoraDescuento.java y coloca el archivo .class resultante en el directorio bin en lugar del directorio actual. Útil para mantener archivos fuente y compilados separados.

Mostrar advertencias:

  javac -Xlint CalculadoraDescuento.java

Muestra advertencias sobre prácticas cuestionables que no son errores pero podrían indicar problemas.

Especificar versión de Java:

  javac -source 17 -target 17 CalculadoraDescuento.java

Compila el código para que sea compatible con Java 17 específicamente.

Ejemplo completo: Proceso de compilación paso a paso

Código fuente — SistemaInventario.java:

  public class SistemaInventario {

    public static void main(String[] args) {

        System.out.println("===== SISTEMA DE INVENTARIO =====");

        // Datos del producto
        String codigoProducto = "PROD-550";
        String nombreProducto = "Teclado Mecánico";
        int stockActual = 75;
        int stockMinimo = 20;

        // Calcular estado
        int diferenciaStock = stockActual - stockMinimo;

        // Mostrar información
        System.out.println("Código: " + codigoProducto);
        System.out.println("Producto: " + nombreProducto);
        System.out.println("Stock actual: " + stockActual + " unidades");
        System.out.println("Stock mínimo: " + stockMinimo + " unidades");
        System.out.println("Diferencia: " + diferenciaStock + " unidades");

        if (stockActual < stockMinimo) {
            System.out.println("ALERTA: Stock bajo mínimo");
        } else {
            System.out.println("Estado: Normal");
        }

    }

}

Paso 1: Escribir el código fuente

Guardamos el código en archivo SistemaInventario.java.

Paso 2: Compilar

  javac SistemaInventario.java

Paso 3: Verificar archivo .class generado

  ls -l
# Muestra:
# SistemaInventario.java
# SistemaInventario.class

Paso 4: Ejecutar

  java SistemaInventario

Salida:

===== SISTEMA DE INVENTARIO =====
Código: PROD-550
Producto: Teclado Mecánico
Stock actual: 75 unidades
Stock mínimo: 20 unidades
Diferencia: 55 unidades
Estado: Normal

Compilación versus ejecución

Es importante distinguir claramente entre errores de compilación y errores de ejecución.

Errores de compilación se detectan por javac antes de que el programa ejecute. Incluyen:

  • Errores de sintaxis (falta punto y coma, llaves desbalanceadas)
  • Errores de tipo (asignar double a int)
  • Variables no declaradas
  • Nombres de clase incorrectos

Si hay errores de compilación, NO se genera archivo .class y el programa NO puede ejecutarse.

Errores de ejecución ocurren cuando el programa se ejecuta. El código compila correctamente pero algo sale mal durante la ejecución. Por ejemplo, dividir por cero causa error de ejecución, pero el compilador no puede detectarlo porque depende de los valores en tiempo de ejecución.

  public class ErrorEjecucion {
    public static void main(String[] args) {
        int dividendo = 100;
        int divisor = 0;
        int resultado = dividendo / divisor; // Error en ejecución, no compilación
        System.out.println(resultado);
    }
}

Este código compila sin errores porque sintácticamente es correcto. Pero cuando se ejecuta, lanza ArithmeticException porque no se puede dividir por cero.

Buenas prácticas de compilación

Compilar frecuentemente. No esperar a terminar todo el programa para compilar. Compilar después de agregar cada pocas líneas de código permite detectar errores temprano cuando son más fáciles de identificar y corregir.

Leer mensajes de error completos. No solo mirar el número de línea. Leer la descripción completa del error para entender qué está mal.

Corregir errores de arriba hacia abajo. Los primeros errores frecuentemente causan errores en cascada. Corregir el primer error y recompilar puede hacer que múltiples errores subsecuentes desaparezcan.

Mantener archivos .java y .class organizados. Considerar usar estructura de directorios que separe código fuente de archivos compilados:

proyecto/
├── src/
│   └── SistemaInventario.java
└── bin/
    └── SistemaInventario.class

Compilar con:

  javac -d bin src/SistemaInventario.java

No versionar archivos .class. En sistemas de control de versiones como Git, solo versionar archivos .java, nunca archivos .class. Los archivos .class son generados y deben estar en .gitignore.

La compilación es el puente entre el código que escribimos y el programa que ejecuta. Comprender este proceso, saber interpretar mensajes del compilador, y desarrollar buenos hábitos de compilación frecuente facilita enormemente el desarrollo de programas Java correctos y reduce el tiempo dedicado a depuración de errores sintácticos que el compilador habría detectado si se hubiera invocado oportunamente.

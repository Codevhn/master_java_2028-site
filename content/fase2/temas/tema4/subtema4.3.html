4.3 Relación entre archivo .java y clase

La relación entre un archivo .java y la clase que contiene es uno de los aspectos fundamentales de Java que debe comprenderse desde el principio. Esta relación no es arbitraria sino que sigue reglas estrictas diseñadas para mantener el código organizado, fácil de encontrar, y libre de ambigüedades. Comprender exactamente cómo se relacionan estos dos elementos, qué reglas rigen esta relación, y por qué estas reglas existen es esencial para escribir programas Java correctos y evitar errores de compilación comunes.

La regla fundamental

En Java, el nombre del archivo .java DEBE coincidir exactamente con el nombre de la clase que contiene.

Esta es una regla absoluta y obligatoria. No es una sugerencia o recomendación, es un requisito técnico del lenguaje.

Ejemplo correcto:

// Archivo: CalculadoraDescuento.java

class CalculadoraDescuento {
    public static void main(String[] args) {
        double precioBase = 1000.00;
        double descuento = 150.00;
        double precioFinal = precioBase - descuento;

        System.out.println("Precio original: Lps " + precioBase);
        System.out.println("Descuento: Lps " + descuento);
        System.out.println("Precio final: Lps " + precioFinal);
    }
}

Correspondencia:

  • Nombre del archivo: CalculadoraDescuento.java
  • Nombre de la clase: CalculadoraDescuento
  • ✓ Coinciden exactamente (incluyendo mayúsculas/minúsculas)

¿Por qué existe esta regla?

Razón 1: Facilita encontrar código

Si necesita encontrar el código de la clase CalculadoraDescuento, solo busca el archivo CalculadoraDescuento.java. No hay ambigüedad.

Razón 2: Evita conflictos

Garantiza que cada clase tiene un "hogar" único y claro en el sistema de archivos.

Razón 3: Simplifica la compilación

El compilador javac sabe exactamente dónde buscar el código de cada clase.

Qué sucede si los nombres NO coinciden

Ejemplo incorrecto:

// Archivo: MiPrograma.java (nombre del archivo)

class CalculadoraVentas { // Nombre de la clase (diferente)
    public static void main(String[] args) {
        System.out.println("Intentando ejecutar");
    }
}

Intento de compilar:

javac MiPrograma.java

Error de compilación:

MiPrograma.java:3: error: class CalculadoraVentas is public, should be declared in a file named CalculadoraVentas.java
class CalculadoraVentas {
^
1 error

El compilador es claro: La clase CalculadoraVentas debe estar en un archivo llamado CalculadoraVentas.java.

No compilará hasta que se corrija.

Solución a nombres que no coinciden

Opción 1: Renombrar el archivo para que coincida con la clase

# Renombrar archivo
mv MiPrograma.java CalculadoraVentas.java

# Ahora compilar
javac CalculadoraVentas.java
# ✓ Compila exitosamente

Opción 2: Renombrar la clase para que coincida con el archivo

// Archivo: MiPrograma.java

class MiPrograma { // Cambiar nombre de clase
    public static void main(String[] args) {
        System.out.println("Ahora funciona");
    }
}
javac MiPrograma.java
# ✓ Compila exitosamente

Sensibilidad a mayúsculas y minúsculas

Java es extremadamente sensible a mayúsculas y minúsculas. Calculadora y calculadora son nombres completamente diferentes.

Ejemplo de problema:

// Archivo: calculadora.java (todo minúsculas)

class Calculadora { // Primera letra mayúscula
    public static void main(String[] args) {
        System.out.println("Hola");
    }
}

En sistemas Linux/macOS:

javac calculadora.java
# Error: nombres no coinciden

En Windows:

  • Puede funcionar (Windows ignora mayúsculas/minúsculas en nombres de archivo)
  • PERO esto es peligroso porque el código no funcionará en otros sistemas

Solución correcta:

# Renombrar archivo para que coincida exactamente
mv calculadora.java Calculadora.java

# Ahora compilar
javac Calculadora.java
# ✓ Funciona en todos los sistemas

Correspondencia uno a uno

Cada archivo .java contiene exactamente UNA clase.

Esta es la práctica estándar y recomendada en Java profesional.

Estructura recomendada:

proyecto/
├── CalculadoraDescuento.java → contiene class CalculadoraDescuento
├── CalculadoraSalario.java   → contiene class CalculadoraSalario
└── ReporteVentas.java        → contiene class ReporteVentas

Cada archivo tiene su propia clase, nombres coincidentes.

Ejemplo completo: Sistema de facturación

Archivo: CalculadoraFactura.java

// Sistema de cálculo de factura con impuestos

class CalculadoraFactura {
    public static void main(String[] args) {
        // Datos de productos
        double precioProducto1 = 250.00;
        double precioProducto2 = 480.00;
        double precioProducto3 = 125.00;

        // Calcular subtotal
        double subtotal = precioProducto1 + precioProducto2 + precioProducto3;

        // Calcular impuesto (15%)
        double porcentajeImpuesto = 15.00;
        double montoImpuesto = subtotal * porcentajeImpuesto / 100;

        // Calcular total
        double totalFactura = subtotal + montoImpuesto;

        // Mostrar factura
        System.out.println("========== FACTURA ==========");
        System.out.println("Producto 1: Lps " + precioProducto1);
        System.out.println("Producto 2: Lps " + precioProducto2);
        System.out.println("Producto 3: Lps " + precioProducto3);
        System.out.println("-----------------------------");
        System.out.println("Subtotal: Lps " + subtotal);
        System.out.println("Impuesto (15%): Lps " + montoImpuesto);
        System.out.println("=============================");
        System.out.println("TOTAL: Lps " + totalFactura);
    }
}

Relación archivo-clase:

  • Archivo: CalculadoraFactura.java
  • Clase: CalculadoraFactura
  • ✓ Coinciden perfectamente

Compilar y ejecutar:

javac CalculadoraFactura.java
java CalculadoraFactura

Salida:

========== FACTURA ==========
Producto 1: Lps 250.0
Producto 2: Lps 480.0
Producto 3: Lps 125.0
-----------------------------
Subtotal: Lps 855.0
Impuesto (15%): Lps 128.25
=============================
TOTAL: Lps 983.25

Del archivo .java a la ejecución

Proceso completo:

1. ARCHIVO FUENTE
CalculadoraFactura.java
(contiene class CalculadoraFactura)
↓

2. COMPILACIÓN
javac CalculadoraFactura.java
↓
Compilador verifica:
- Sintaxis correcta
- Nombres coinciden
- Reglas de Java seguidas
↓
Genera:
CalculadoraFactura.class
↓

3. EJECUCIÓN
java CalculadoraFactura
↓
JVM:
- Carga CalculadoraFactura.class
- Busca método main
- Ejecuta el programa

Verificar la relación archivo-clase

Después de escribir un programa, verifique que todo está correcto:

Paso 1: Verificar nombre del archivo

# Ver archivos en el directorio
ls -l # macOS/Linux
dir   # Windows

Confirme que el archivo termina en .java y el nombre es correcto.

Paso 2: Abrir archivo y verificar nombre de clase

class NombreClase { // ¿Este nombre coincide con el archivo?
    // ...
}

Paso 3: Intentar compilar

javac NombreArchivo.java

Si compila sin errores → nombres coinciden ✓

Si da error de nombres → corregir archivo o clase

Ejemplo: Calculadora de nómina

Archivo: CalculadoraNomina.java

// Calculadora de nómina semanal para empleados

class CalculadoraNomina {

Errores comunes y sus soluciones

Error 1: Clase y archivo con nombres diferentes

// Archivo: Programa.java
class ReporteVentas {
    // ✗ Nombres no coinciden
}

Solución: Renombrar archivo a ReporteVentas.java

Error 2: Mayúsculas/minúsculas inconsistentes

// Archivo: calculadoraventas.java
class CalculadoraVentas {
    // ✗ Mayúsculas no coinciden
}

Solución: Renombrar archivo a CalculadoraVentas.java

Error 3: Espacios en nombres

// Archivo: Calculadora Ventas.java (con espacio)
class CalculadoraVentas {
    // ✗ Espacio en nombre de archivo
}

Solución: Renombrar a CalculadoraVentas.java (sin espacios)

Error 4: Extensión incorrecta

Archivo: CalculadoraVentas.txt

Solución: Cambiar extensión a .java

Buenas prácticas

1. Siempre verificar coincidencia antes de compilar

Antes de ejecutar javac, verifique:

  • Nombre del archivo
  • Nombre de la clase dentro del archivo
  • ¿Coinciden exactamente?

2. Usar nombres descriptivos

// ✓ Nombre descriptivo y claro
class CalculadoraImpuestoVentas {
    // ...
}
// ✗ Nombre genérico y confuso
class Calc {
    // ...
}

3. Seguir convención PascalCase

// ✓ Correcto: primera letra de cada palabra en mayúscula
class SistemaGestionInventario {
    // ...
}
// ✗ No recomendado
class sistema_gestion_inventario {
    // ...
}

4. Un archivo = una clase

No intente poner múltiples programas en un archivo. Mantenga simple:

proyecto/
├── CalculadoraDescuento.java → Una clase
├── ReporteVentas.java        → Una clase
└── SistemaNomina.java        → Una clase

Ejemplo final: Control de inventario

Archivo: ControlInventario.java

// Sistema de control de inventario básico

class ControlInventario {
    public static void main(String[] args) {
        // Inventario inicial
        int stockInicial = 100;

        // Movimientos del día
        int entradasDia = 25;
        int salidasDia = 40;

        // Calcular stock final
        int stockFinal = stockInicial + entradasDia - salidasDia;

        // Mostrar reporte
        System.out.println("===== CONTROL DE INVENTARIO =====");
        System.out.println("Stock inicial: " + stockInicial + " unidades");
        System.out.println("Entradas: +" + entradasDia + " unidades");
        System.out.println("Salidas: -" + salidasDia + " unidades");
        System.out.println("---------------------------------");
        System.out.println("Stock final: " + stockFinal + " unidades");
        System.out.println("=================================");
    }
}

Verificación de correspondencia:

  • ✓ Archivo: ControlInventario.java
  • ✓ Clase: ControlInventario
  • ✓ Coincidencia perfecta

La relación entre archivo .java y clase es simple pero estricta: deben tener el mismo nombre, exactamente. Esta regla, aunque puede parecer restrictiva al principio, en realidad simplifica enormemente la organización del código y hace que sea fácil encontrar y mantener programas. Seguir esta regla desde el inicio crea buenos hábitos que facilitarán el desarrollo a medida que los programas se vuelvan más complejos.

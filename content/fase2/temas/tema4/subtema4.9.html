4.9 Uso de comentarios en la estructura del programa

Los comentarios son anotaciones en el código que explican qué hace el programa, por qué se tomaron ciertas decisiones, o qué representa una sección específica. No son instrucciones ejecutables sino notas para humanos que leen el código. El compilador ignora completamente los comentarios, pero para los desarrolladores son herramientas esenciales para documentar, aclarar y organizar el código. Comprender qué son los comentarios, cuándo usarlos, cómo escribirlos correctamente, y qué buenas prácticas seguir es fundamental para escribir código profesional que otros (y nuestro yo futuro) puedan entender fácilmente.

¿Qué son los comentarios?

Los comentarios son texto dentro del código fuente que:

  • El compilador ignora completamente
  • NO se ejecutan
  • Sirven para explicar el código a humanos
  • Ayudan a documentar el programa

Analogía:

Los comentarios son como notas al margen en un libro:

  • El contenido principal (código) es lo que se ejecuta
  • Las notas (comentarios) explican el contenido
  • Puede leer el libro sin las notas, pero las notas ayudan a entender mejor

Tipos de comentarios en Java

Java tiene tres tipos de comentarios:

1. Comentarios de una sola línea (//)
2. Comentarios de múltiples líneas (/* */)
3. Comentarios de documentación (/** */) (veremos más adelante)

Por ahora nos enfocaremos en los dos primeros.

Comentarios de una sola línea (//)

Sintaxis: Todo lo que está después de // hasta el final de la línea es un comentario.

Ejemplo:

// Este es un comentario de una sola línea
double precio = 100.0; // Esto también es un comentario

Características:

  • Comienza con //
  • Termina automáticamente al final de la línea
  • Útil para comentarios breves o explicaciones inline

Ejemplo con comentarios de una línea:

public class CalculadoraDescuento {

    public static void main(String[] args) {

        // Precio original del producto
        double precioOriginal = 850.00;

        // Porcentaje de descuento aplicable
        double porcentajeDescuento = 15.0;

        // Calcular el monto del descuento
        double montoDescuento = precioOriginal * porcentajeDescuento / 100.0;

        // Calcular el precio final
        double precioFinal = precioOriginal - montoDescuento;

        // Mostrar resultados
        System.out.println("Precio original: Lps " + precioOriginal);
        System.out.println("Descuento (15%): Lps " + montoDescuento);
        System.out.println("Precio final: Lps " + precioFinal);

    }

}

Los comentarios explican qué hace cada sección del código.

Comentarios de múltiples líneas (/* */)

Sintaxis: Todo lo que está entre /* y */ es un comentario, puede ocupar múltiples líneas.

Ejemplo:

/*
Este es un comentario
que ocupa múltiples líneas.
Puede escribir varias líneas de explicación.
*/
double precio = 100.0;

Características:

  • Comienza con /*
  • Termina con */
  • Puede ocupar una o muchas líneas
  • Útil para bloques grandes de explicación

Ejemplo con comentarios de múltiples líneas:

public class CalculadoraNomina {

    public static void main(String[] args) {

        /*
         * Sistema de cálculo de nómina
         * Calcula salario semanal incluyendo horas extras
         * Tarifa normal: Lps 125 por hora
         * Tarifa horas extras: Lps 187.50 por hora (1.5x)
         */

        double horasNormales = 40.0;
        double horasExtras = 5.0;
        double tarifaNormal = 125.00;

        /* Calcular tarifa de horas extras (1.5 veces la tarifa normal) */
        double tarifaExtras = tarifaNormal * 1.5;

        /* Calcular salarios */
        double salarioNormal = horasNormales * tarifaNormal;
        double salarioExtras = horasExtras * tarifaExtras;
        double salarioTotal = salarioNormal + salarioExtras;

        System.out.println("===== NÓMINA SEMANAL =====");
        System.out.println("Horas normales: " + horasNormales + " hrs");
        System.out.println("Horas extras: " + horasExtras + " hrs");
        System.out.println("-------------------------");
        System.out.println("Salario normal: Lps " + salarioNormal);
        System.out.println("Salario extras: Lps " + salarioExtras);
        System.out.println("=========================");
        System.out.println("TOTAL: Lps " + salarioTotal);

    }

}

Diferencia entre // y /* */

Característica // /* */
Líneas Una sola Una o múltiples
Uso típico Comentarios breves Bloques de explicación
Finaliza Al final de línea Con */

Ejemplo comparativo:

// Esto es un comentario corto de una línea

/*
Esto es un comentario más largo
que requiere varias líneas
para explicar algo complejo
*/

El compilador ignora los comentarios

Código con comentarios:

double precio = 100.0;               // Declarar precio
// double descuento = 20.0;          // Esta línea está comentada
double total = precio;               // Sin descuento

Lo que el compilador "ve":

double precio = 100.0;
double total = precio;

El compilador elimina completamente todos los comentarios antes de compilar.

Comentar código temporalmente

Los comentarios son útiles para desactivar temporalmente código sin borrarlo:

public class Prueba {

    public static void main(String[] args) {

        double precio = 500.0;

        // Temporalmente desactivado
        // double descuento = precio * 0.1;
        // precio = precio - descuento;

        System.out.println("Precio: " + precio);

    }

}

El código comentado no se ejecuta, pero está ahí para referencia o para reactivarlo después.

Comentarios dentro de líneas de código

Puede poner comentarios al final de líneas de código:

double precioBase = 1000.00;             // Precio antes de impuestos
double tasaImpuesto = 15.0;              // Tasa del 15%
double impuesto = precioBase * 0.15;    // Calcular impuesto
double total = precioBase + impuesto;   // Sumar precio e impuesto

¿Cuándo usar comentarios?

USE comentarios para:

1. Explicar secciones del programa

// ===== SECCIÓN: CÁLCULO DE IMPUESTOS =====
double impuesto = subtotal * 0.15;

2. Aclarar cálculos complejos

// Fórmula: interés = capital * tasa * tiempo / 100
double interes = capital * tasaAnual * anios / 100.0;

3. Documentar decisiones importantes

// Usamos 24.50 como tasa de cambio fija según acuerdo del 15/03/2026
double tasaCambio = 24.50;

4. Marcar secciones de código

// ===== ENTRADA DE DATOS =====
double precio = 500.0;

// ===== CÁLCULOS =====
double descuento = precio * 0.1;

// ===== RESULTADOS =====
System.out.println(descuento);

NO use comentarios para:

1. Explicar código obvio

// ✗ Comentario innecesario
double precio = 100.0; // Declarar variable precio con valor 100.0

El código ya es claro por sí mismo.

2. Comentar cada línea

// ✗ Exceso de comentarios
double x = 10.0; // Declarar x
double y = 20.0; // Declarar y
double z = x + y; // Sumar x e y

Demasiados comentarios hacen el código difícil de leer.

3. Información desactualizada

// ✗ Comentario desactualizado
// Tasa de cambio: 20.00 Lps/USD
double tasaCambio = 24.50; // El comentario no coincide con el código

Buenas prácticas con comentarios

1. Usar nombres de variables descriptivos en lugar de comentarios

// ✗ Necesita comentario
double t = 24.50; // Tasa de cambio

// ✓ No necesita comentario (nombre claro)
double tasaCambioDolarLempira = 24.50;

2. Comentar el "por qué", no el "qué"

// ✗ Comenta lo obvio
double descuento = precio * 0.1; // Multiplicar precio por 0.1

// ✓ Explica la razón
double descuento = precio * 0.1; // 10% de descuento por compra mayorista

3. Mantener comentarios actualizados

Si cambia el código, actualice los comentarios relacionados.

4. Usar comentarios para organizar secciones

public class SistemaVentas {

    public static void main(String[] args) {

        // ===== CONFIGURACIÓN INICIAL =====
        double tasaImpuesto = 15.0;

        // ===== DATOS DE VENTA =====
        double precioProducto = 500.0;
        int cantidad = 3;

        // ===== CÁLCULOS =====
        double subtotal = precioProducto * cantidad;
        double impuesto = subtotal * tasaImpuesto / 100.0;
        double total = subtotal + impuesto;

        // ===== MOSTRAR RESULTADOS =====
        System.out.println("Subtotal: Lps " + subtotal);
        System.out.println("Impuesto: Lps " + impuesto);
        System.out.println("Total: Lps " + total);

    }

}

Ejemplo completo: Sistema de facturación con comentarios bien usados

public class SistemaFacturacion {

    public static void main(String[] args) {

        /*
         * Sistema de Facturación - Versión 1.0
         * Calcula factura con descuento por volumen e impuesto
         * Autor: Equipo de Desarrollo
         */

        System.out.println("===== SISTEMA DE FACTURACIÓN =====");
        System.out.println();

        // ===== DATOS DEL CLIENTE =====
        String nombreCliente = "Comercial Hernández";

        // ===== DATOS DE PRODUCTOS =====
        double precioUnitario = 125.00;
        int cantidadComprada = 50;

        // ===== CÁLCULO DE SUBTOTAL =====
        double subtotal = precioUnitario * cantidadComprada;

        // ===== DESCUENTO POR VOLUMEN =====
        // Aplicar 10% de descuento si compra 50 unidades o más
        double porcentajeDescuento = 10.0;
        double montoDescuento = subtotal * porcentajeDescuento / 100.0;

        // Subtotal después de descuento
        double subtotalConDescuento = subtotal - montoDescuento;

        // ===== CÁLCULO DE IMPUESTO =====
        // Tasa de ISV (Impuesto sobre Ventas) en Honduras
        double tasaImpuesto = 15.0;
        double montoImpuesto = subtotalConDescuento * tasaImpuesto / 100.0;

        // ===== TOTAL FINAL =====
        double totalFactura = subtotalConDescuento + montoImpuesto;

        // ===== MOSTRAR FACTURA =====
        System.out.println("Cliente: " + nombreCliente);
        System.out.println("Cantidad: " + cantidadComprada + " unidades");
        System.out.println("Precio unitario: Lps " + precioUnitario);
        System.out.println("----------------------------------");
        System.out.println("Subtotal: Lps " + subtotal);
        System.out.println("Descuento (10%): -Lps " + montoDescuento);
        System.out.println("Subtotal con desc.: Lps " + subtotalConDescuento);
        System.out.println("Impuesto (15%): Lps " + montoImpuesto);
        System.out.println("==================================");
        System.out.println("TOTAL A PAGAR: Lps " + totalFactura);

    }

}

Salida:

===== SISTEMA DE FACTURACIÓN =====

Cliente: Comercial Hernández
Cantidad: 50 unidades
Precio unitario: Lps 125.0
----------------------------------
Subtotal: Lps 6250.0
Descuento (10%): -Lps 625.0
Subtotal con desc.: Lps 5625.0
Impuesto (15%): Lps 843.75
==================================
TOTAL A PAGAR: Lps 6468.75

Comentarios anidados (CUIDADO)

NO puede anidar comentarios de múltiples líneas:

/*
Este es un comentario
/* Intentando anidar otro comentario */ // ✗ Causa error
*/

Error: El primer */ cierra el comentario, dejando código inválido después.

Solución: Usar solo un nivel de comentarios /* */ o usar // dentro:

/*
Este es un comentario
// Puedo usar // dentro de /* */
*/

Comentarios y depuración

Los comentarios son útiles para depurar (encontrar errores):

Técnica: Comentar secciones para aislar errores

public class Prueba {

    public static void main(String[] args) {

        double precio = 500.0;

        // Comentar temporalmente para probar
        // double descuento = precio * 0.1;
        // precio = precio - descuento;

        System.out.println("Precio: " + precio);

        // Una vez que funcione, descomentar

    }

}

Ejemplo final: Control de inventario con comentarios profesionales

/*
 * Sistema de Control de Inventario
 * Registra movimientos diarios de entrada y salida
 * Versión: 1.0
 */

public class ControlInventario {

    public static void main(String[] args) {

        System.out.println("===== CONTROL DE INVENTARIO =====");
        System.out.println();

        // ===== INVENTARIO INICIAL =====
        // Stock al inicio del día
        int stockInicial = 200;

        // ===== MOVIMIENTOS DEL DÍA =====

        // Turno mañana
        int entradasMañana = 40; // Recepción de proveedor
        int salidasMañana = 55;  // Ventas matutinas

        // Turno tarde
        int entradasTarde = 25;  // Devoluciones de clientes
        int salidasTarde = 48;   // Ventas vespertinas

        // ===== CÁLCULO DE TOTALES =====
        int totalEntradas = entradasMañana + entradasTarde;
        int totalSalidas = salidasMañana + salidasTarde;

        // Calcular stock final del día
        int stockFinal = stockInicial + totalEntradas - totalSalidas;

        // ===== REPORTE DEL DÍA =====
        System.out.println("Stock inicial: " + stockInicial + " unidades");
        System.out.println("----------------------------------");
        System.out.println("MOVIMIENTOS:");
        System.out.println("  Entradas mañana: +" + entradasMañana);
        System.out.println("  Salidas mañana: -" + salidasMañana);
        System.out.println("  Entradas tarde: +" + entradasTarde);
        System.out.println("  Salidas tarde: -" + salidasTarde);
        System.out.println("----------------------------------");
        System.out.println("Total entradas: +" + totalEntradas);
        System.out.println("Total salidas: -" + totalSalidas);
        System.out.println("==================================");
        System.out.println("Stock final: " + stockFinal + " unidades");

        // Advertencia si stock bajo
        // En versión futura se agregará alerta automática si stock < 50

    }

}

Comentarios bien usados:

  • ✓ Explican secciones del código
  • ✓ Aclaran decisiones de negocio
  • ✓ Organizan visualmente el programa
  • ✓ Documentan funcionalidad futura
  • ✓ No comentan lo obvio

Los comentarios son herramientas esenciales para documentar y explicar el código, facilitando su comprensión tanto para otros desarrolladores como para nosotros mismos cuando revisamos código antiguo. Usados correctamente, hacen el código más mantenible y profesional. Usados en exceso o incorrectamente, pueden hacer el código más confuso. La clave es comentar el "por qué" de las decisiones, organizar visualmente el código, y mantener los comentarios actualizados con los cambios del programa.

4.13: Flujo de ejecución de un programa Java

Cuando ejecutamos un programa Java escribiendo java NombreClase en la línea de comandos, se desencadena una secuencia precisa de eventos que transforma el bytecode estático en un archivo .class en un programa en ejecución. Comprender este flujo de ejecución, desde el momento en que la JVM inicia hasta que el programa termina, es fundamental para entender cómo Java ejecuta nuestro código y por qué se ejecuta en el orden que lo hace. Este conocimiento también es crucial para depurar programas cuando el comportamiento no es el esperado.

El flujo de ejecución en Java es determinístico y secuencial en su forma más básica. Las instrucciones dentro del método main se ejecutan una tras otra, de arriba hacia abajo, en el orden exacto en que fueron escritas. Esta secuencialidad simple es la base sobre la cual se construyen comportamientos más complejos. Aunque más adelante aprenderemos estructuras de control que permiten saltar, repetir o condicionar la ejecución de código, por ahora el flujo es lineal y predecible.

Visualizar mentalmente cómo la JVM "lee" y ejecuta el código línea por línea ayuda a predecir qué hará un programa antes de ejecutarlo. Esta habilidad de "ejecutar el código mentalmente" es una de las diferencias clave entre programadores principiantes y experimentados. Los principiantes ejecutan el programa y observan qué sucede. Los programadores con experiencia pueden predecir con precisión qué hará el código antes de ejecutarlo porque comprenden el flujo de ejecución.

Inicio de la ejecución: comando java

La ejecución comienza cuando escribimos el comando java seguido del nombre de la clase en la terminal:

  java CalculadoraDescuento

Este comando simple desencadena una serie de acciones complejas dentro de la JVM. Primero, la JVM inicia y carga su entorno de ejecución en memoria. Luego, el launcher de Java busca el archivo CalculadoraDescuento.class en el classpath. Si el archivo no existe, la JVM reporta un error y termina. Si el archivo existe, la JVM lo carga en memoria.

Una vez cargado el archivo .class, la JVM examina el bytecode para verificar que es válido y no ha sido corrompido. Este proceso de verificación asegura que el bytecode sigue las reglas de la especificación de la JVM y no contiene instrucciones que podrían comprometer la seguridad o estabilidad del sistema. Si la verificación falla, la JVM rechaza el bytecode y termina con error.

Después de verificar el bytecode, la JVM busca el método main con la firma exacta public static void main(String[] args). Este método es especial porque la especificación de la JVM define que la ejecución del programa comienza ahí. Si la clase no tiene método main o si la firma no coincide exactamente, la JVM reporta error y termina.

  public class EjemploFlujo {
    public static void main(String[] args) {
        // La ejecución comienza aquí
        System.out.println("Inicio del programa");
    }
}

Cuando ejecutamos java EjemploFlujo, la JVM encuentra el método main y transfiere el control a la primera instrucción dentro de ese método. Esa primera instrucción es System.out.println("Inicio del programa");. La JVM ejecuta esta instrucción, que resulta en texto siendo escrito a la salida estándar, y luego avanza a la siguiente instrucción.

Ejecución secuencial de instrucciones

Una vez que la JVM comienza a ejecutar el método main, procesa las instrucciones una por una, en el orden exacto en que aparecen en el código fuente, de arriba hacia abajo.

  public class FlujoSecuencial {
    public static void main(String[] args) {

        System.out.println("Instrucción 1");
        System.out.println("Instrucción 2");
        System.out.println("Instrucción 3");
        System.out.println("Instrucción 4");

    }
}

Orden de ejecución:

  1. JVM ejecuta System.out.println("Instrucción 1"); → Muestra "Instrucción 1"
  2. JVM ejecuta System.out.println("Instrucción 2"); → Muestra "Instrucción 2"
  3. JVM ejecuta System.out.println("Instrucción 3"); → Muestra "Instrucción 3"
  4. JVM ejecuta System.out.println("Instrucción 4"); → Muestra "Instrucción 4"
  5. No hay más instrucciones, el método main termina
  6. El programa termina

Salida:

Instrucción 1
Instrucción 2
Instrucción 3
Instrucción 4

La salida siempre será en este orden, nunca variará. No es posible que "Instrucción 3" se ejecute antes que "Instrucción 2" porque la JVM procesa las instrucciones secuencialmente. Esta predictibilidad es fundamental para razonar sobre programas.

Declaraciones y asignaciones en el flujo

Las declaraciones de variables y asignaciones también se ejecutan secuencialmente. Una variable solo existe y contiene un valor después de que su declaración ha sido ejecutada.

  public class FlujoVariables {
    public static void main(String[] args) {

        double precio = 100.0;
        System.out.println("Precio declarado: " + precio);

        double descuento = 15.0;
        System.out.println("Descuento declarado: " + descuento);

        double precioFinal = precio - descuento;
        System.out.println("Precio final calculado: " + precioFinal);

    }
}

Flujo paso a paso:

  1. JVM ejecuta double precio = 100.0;
    • Crea variable precio en memoria
    • Almacena valor 100.0 en esa variable
  2. JVM ejecuta System.out.println("Precio declarado: " + precio);
    • Lee valor de precio (100.0)
    • Construye string "Precio declarado: 100.0"
    • Muestra en pantalla
  3. JVM ejecuta double descuento = 15.0;
    • Crea variable descuento en memoria
    • Almacena valor 15.0 en esa variable
  4. JVM ejecuta System.out.println("Descuento declarado: " + descuento);
    • Lee valor de descuento (15.0)
    • Muestra "Descuento declarado: 15.0"
  5. JVM ejecuta double precioFinal = precio - descuento;
    • Lee valor de precio (100.0)
    • Lee valor de descuento (15.0)
    • Calcula 100.0 - 15.0 = 85.0
    • Crea variable precioFinal y almacena 85.0
  6. JVM ejecuta el último System.out.println → Muestra "Precio final calculado: 85.0"
  7. No hay más instrucciones, método termina, programa termina

Salida:

Precio declarado: 100.0
Descuento declarado: 15.0
Precio final calculado: 85.0

Cada paso depende de que los pasos anteriores se hayan ejecutado. No podríamos calcular precioFinal antes de que precio y descuento hayan sido declarados y asignados. El orden importa.

Dependencias entre instrucciones

El orden de las instrucciones es crítico cuando hay dependencias entre ellas. Una instrucción que usa el resultado de otra debe venir después de la instrucción que produce ese resultado.

  public class OrdenCorrecto {
    public static void main(String[] args) {

        // Correcto: declaramos antes de usar
        double salarioBase = 10000.00;
        double bono = salarioBase * 0.10;
        double salarioTotal = salarioBase + bono;

        System.out.println("Salario total: " + salarioTotal);

    }
}

Este código funciona porque cada variable se declara antes de ser usada. salarioBase se declara primero, luego bono la usa para calcular su valor, y finalmente salarioTotal usa ambas variables anteriores.

Si invertimos el orden, el código no compila:

  public class OrdenIncorrecto {
    public static void main(String[] args) {

        // Error: intentamos usar salarioBase antes de declararla
        double bono = salarioBase * 0.10; // Error de compilación
        double salarioBase = 10000.00;

    }
}

Error de compilación:

error: cannot find symbol
double bono = salarioBase * 0.10;
              ^
symbol:   variable salarioBase
location: class OrdenIncorrecto

El compilador detecta que salarioBase se usa antes de ser declarada y rechaza el código. La JVM nunca permitirá ejecutar código donde las dependencias están en orden incorrecto.

Ejemplo de flujo completo: Calculadora de nómina

  public class CalculadoraNomina {

    public static void main(String[] args) {

        System.out.println("===== CALCULADORA DE NÓMINA =====");

        // Paso 1: Declarar datos de entrada
        String nombreEmpleado = "Ana Martínez";
        double horasTrabajadas = 45.0;
        double tarifaPorHora = 150.00;

        // Paso 2: Calcular salario bruto
        double salarioBruto = horasTrabajadas * tarifaPorHora;
        System.out.println("Salario bruto calculado: Lps " + salarioBruto);

        // Paso 3: Calcular deducciones
        double porcentajeSeguroSocial = 3.5;
        double deduccionSeguroSocial = salarioBruto * porcentajeSeguroSocial / 100.0;

        double porcentajeImpuesto = 10.0;
        double deduccionImpuesto = salarioBruto * porcentajeImpuesto / 100.0;

        System.out.println("Deducciones calculadas");

        // Paso 4: Calcular total de deducciones
        double totalDeducciones = deduccionSeguroSocial + deduccionImpuesto;

        // Paso 5: Calcular salario neto
        double salarioNeto = salarioBruto - totalDeducciones;

        // Paso 6: Mostrar resultado final
        System.out.println("-----------------------------");
        System.out.println("Empleado: " + nombreEmpleado);
        System.out.println("Salario bruto: Lps " + salarioBruto);
        System.out.println("Deducciones: Lps " + totalDeducciones);
        System.out.println("SALARIO NETO: Lps " + salarioNeto);

    }

}

Flujo de ejecución paso a paso:

  1. JVM inicia, carga CalculadoraNomina.class
  2. JVM busca y encuentra método main
  3. JVM comienza ejecución en primera línea de main
  4. Ejecuta el System.out.println del título → Muestra encabezado
  5. Ejecuta String nombreEmpleado = "Ana Martínez"; → Crea variable, almacena nombre
  6. Ejecuta double horasTrabajadas = 45.0; → Crea variable, almacena 45.0
  7. Ejecuta double tarifaPorHora = 150.00; → Crea variable, almacena 150.00
  8. Ejecuta double salarioBruto = horasTrabajadas * tarifaPorHora;
    • Lee horasTrabajadas (45.0) y tarifaPorHora (150.00)
    • Calcula 45.0 * 150.00 = 6750.0 y almacena en salarioBruto
  9. Continúa ejecutando cada instrucción en orden...
  10. Llega al final del método main, el programa termina y la JVM finaliza

Salida:

===== CALCULADORA DE NÓMINA =====
Salario bruto calculado: Lps 6750.0
Deducciones calculadas
-----------------------------
Empleado: Ana Martínez
Salario bruto: Lps 6750.0
Deducciones: Lps 911.25
SALARIO NETO: Lps 5838.75

Cada línea de salida aparece exactamente cuando su instrucción System.out.println correspondiente se ejecuta. No hay magia, no hay desorden. El flujo es completamente predecible.

Visualizar el flujo mentalmente

Una habilidad crítica es poder "ejecutar" el código mentalmente antes de ejecutarlo realmente. Esto permite predecir qué hará el programa y detectar errores lógicos antes de compilar.

  public class PracticaFlujo {
    public static void main(String[] args) {

        double valorA = 20.0;
        double valorB = 30.0;
        double suma = valorA + valorB;

        valorA = 40.0;

        System.out.println("Suma: " + suma);
        System.out.println("ValorA: " + valorA);

    }
}

Ejecutando mentalmente:

  1. valorA se crea y se le asigna 20.0
  2. valorB se crea y se le asigna 30.0
  3. suma se calcula como 20.0 + 30.0 = 50.0
  4. valorA cambia a 40.0 (IMPORTANTE: suma ya fue calculada, NO cambia)
  5. Muestra "Suma: 50.0" (no 70.0, porque suma se calculó antes del cambio)
  6. Muestra "ValorA: 40.0"

Salida predicha:

Suma: 50.0
ValorA: 40.0

Este es un caso común donde principiantes esperan que suma sea 70.0 porque ven que valorA es 40.0 al final. Pero suma se calculó cuando valorA era 20.0, y cambiar valorA después no afecta a suma. El flujo secuencial lo aclara: suma = valorA + valorB se ejecutó ANTES de valorA = 40.0.

Terminación del programa

Un programa Java termina cuando el método main completa su última instrucción y retorna el control a la JVM. En este momento, la JVM libera todos los recursos, cierra archivos abiertos, y finaliza el proceso.

  public class EjemploTerminacion {
    public static void main(String[] args) {

        System.out.println("Primera línea");
        System.out.println("Segunda línea");
        System.out.println("Tercera línea");

        // Aquí termina main, por lo tanto termina el programa
    }
}

Después de ejecutar la tercera línea, no hay más instrucciones. El método main termina, y el programa completo termina. La JVM cierra y el control retorna al sistema operativo.

Si colocamos instrucciones después del punto donde el programa debería terminar, el compilador detecta el código inalcanzable:

  public class CodigoInalcanzable {
    public static void main(String[] args) {

        System.out.println("Esta línea se ejecuta");

        return; // Salida explícita de main

        System.out.println("Esta línea nunca se ejecuta"); // Inalcanzable

    }
}

La instrucción return; termina inmediatamente la ejecución de main, por lo tanto el System.out.println que viene después nunca se ejecuta. El compilador incluso detecta esto y reporta advertencia sobre código inalcanzable.

Ejemplo completo: Sistema de conversión de moneda

  public class ConversionMoneda {

    public static void main(String[] args) {

        System.out.println("===== CONVERSIÓN DE DÓLARES A LEMPIRAS =====");
        System.out.println();

        // Entrada: cantidad en dólares
        double cantidadDolares = 250.00;
        System.out.println("Cantidad a convertir: $" + cantidadDolares);

        // Configuración: tasa de cambio
        double tasaCambio = 24.50;
        System.out.println("Tasa de cambio: Lps " + tasaCambio + " por USD");
        System.out.println();

        // Procesamiento: conversión
        double cantidadLempiras = cantidadDolares * tasaCambio;
        System.out.println("Conversión realizada...");

        // Procesamiento: comisión bancaria
        double porcentajeComision = 2.0;
        double montoComision = cantidadLempiras * porcentajeComision / 100.0;
        System.out.println("Comisión calculada...");

        // Resultado final
        double montoFinalRecibir = cantidadLempiras - montoComision;

        // Salida: mostrar resultados
        System.out.println();
        System.out.println("DESGLOSE:");
        System.out.println("  Equivalente en Lps: Lps " + cantidadLempiras);
        System.out.println("  Comisión bancaria:  Lps " + montoComision);
        System.out.println("============================================");
        System.out.println("  MONTO A RECIBIR:   Lps " + montoFinalRecibir);

        System.out.println();
        System.out.println("Transacción completada.");

    }

}

Este programa demuestra claramente el flujo de ejecución: entrada de datos, configuración, procesamiento paso a paso con mensajes de progreso, cálculo de resultado final, y salida de resultados. Cada etapa se ejecuta en orden, y la salida muestra el progreso del programa a medida que avanza por el flujo de ejecución.

Comprender el flujo de ejecución secuencial es fundamental porque establece el modelo mental básico de cómo Java ejecuta programas. Aunque más adelante aprenderemos estructuras de control que permiten alterar este flujo secuencial simple, la ejecución paso a paso, instrucción por instrucción, permanece como el comportamiento fundamental subyacente. Un programa complejo que usa bucles, condicionales, y llamadas a métodos todavía ejecuta instrucciones individuales secuencialmente, solo que el orden puede ser más complejo que simplemente de arriba hacia abajo.

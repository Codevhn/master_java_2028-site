4.8 Sentencias y terminadores de instrucción

En Java, las sentencias son las instrucciones individuales que le dicen al programa qué hacer: declarar variables, realizar cálculos, mostrar resultados, o ejecutar cualquier acción específica. Cada sentencia debe terminar con un símbolo específico que indica al compilador "esta instrucción ha finalizado". Comprender qué es una sentencia, cómo se estructura, por qué debe terminar con punto y coma, y qué errores surgen cuando se omite este terminador es fundamental para escribir código Java sintácticamente correcto. Este subtema explora el concepto de sentencias y el uso del punto y coma como terminador obligatorio.

¿Qué es una sentencia?

Una sentencia (también llamada instrucción) es una unidad completa de ejecución en Java. Es una orden específica que el programa debe realizar.

Ejemplos de sentencias:

double precio = 100.0;                     // Declaración de variable
double descuento = precio * 0.1;           // Cálculo
System.out.println("Total: " + precio);   // Mostrar en pantalla

Cada línea es una sentencia completa.

Analogía:

Una sentencia en Java es como una oración en español:

  • Tiene un inicio
  • Tiene un final (punto y coma ;)
  • Expresa una idea completa
  • "Declara la variable precio con valor 100.0;"

El punto y coma ; como terminador

En Java, cada sentencia debe terminar con punto y coma ;

Regla fundamental: El punto y coma indica al compilador "esta instrucción terminó, comienza la siguiente".

Ejemplo correcto:

double valorA = 50.0;            // ✓ Termina con ;
double valorB = 30.0;            // ✓ Termina con ;
double suma = valorA + valorB;   // ✓ Termina con ;

Ejemplo incorrecto:

double valorA = 50.0  // ✗ Falta ;
double valorB = 30.0  // ✗ Falta ;

Error de compilación:

error: ';' expected
double valorA = 50.0
^

El compilador indica exactamente dónde falta el punto y coma.

Tipos de sentencias básicas

1. Declaración de variable

Crea una variable con un nombre y tipo:

int cantidad;
double precio;
String producto;

Cada declaración termina con ;

2. Declaración con inicialización

Crea una variable y le asigna un valor inicial:

int cantidad = 10;
double precio = 250.50;
String producto = "Laptop";

Cada una es una sentencia completa con ;

3. Asignación

Asigna un valor a una variable ya existente:

cantidad = 20;
precio = 300.00;
producto = "Monitor";

4. Expresión de cálculo

Realiza una operación y guarda el resultado:

double total = precio * cantidad;
double descuento = total * 0.15;
double precioFinal = total - descuento;

5. Llamada a método (como println)

Invoca una acción específica:

System.out.println("Hola mundo");
System.out.println("Precio: " + precio);

Ejemplo completo con diferentes tipos de sentencias

public class EjemploSentencias {

    public static void main(String[] args) {

        // Sentencia 1: Declaración con inicialización
        double precioUnitario = 125.00;

        // Sentencia 2: Declaración con inicialización
        int cantidadProductos = 5;

        // Sentencia 3: Cálculo (expresión)
        double subtotal = precioUnitario * cantidadProductos;

        // Sentencia 4: Declaración de variable
        double tasaImpuesto;

        // Sentencia 5: Asignación
        tasaImpuesto = 15.0;

        // Sentencia 6: Cálculo
        double montoImpuesto = subtotal * tasaImpuesto / 100.0;

        // Sentencia 7: Cálculo
        double totalFactura = subtotal + montoImpuesto;

        // Sentencia 8: Llamada a método
        System.out.println("===== FACTURA =====");

        // Sentencia 9: Llamada a método
        System.out.println("Subtotal: Lps " + subtotal);

        // Sentencia 10: Llamada a método
        System.out.println("Impuesto: Lps " + montoImpuesto);

        // Sentencia 11: Llamada a método
        System.out.println("Total: Lps " + totalFactura);

    }

}

Todas las sentencias terminan con ;

¿Por qué es obligatorio el punto y coma?

Razón 1: Delimita instrucciones

Sin el punto y coma, el compilador no sabría dónde termina una instrucción y comienza la siguiente:

// Sin punto y coma (ambiguo)
double x = 10 double y = 20 // ¿Es una instrucción o dos?

// Con punto y coma (claro)
double x = 10; // Primera instrucción
double y = 20; // Segunda instrucción

Razón 2: Permite instrucciones en múltiples líneas

Una sentencia puede ocupar varias líneas, el ; indica que terminó:

double resultado = (precioProducto1 + precioProducto2 +
                   precioProducto3 + precioProducto4) *
                   tasaImpuesto / 100.0; // ✓ Una sola sentencia

El compilador sabe que es UNA sentencia porque solo hay un ; al final.

Razón 3: Permite múltiples instrucciones en una línea (aunque no recomendado)

double x = 10; double y = 20; double z = 30; // Técnicamente válido

Aunque esto compila, NO es recomendado. Mejor práctica: una sentencia por línea.

Errores comunes con punto y coma

Error 1: Olvidar el punto y coma

double precio = 100.0  // ✗ Falta ;
double descuento = 10.0; // ✗ Falta ; en línea anterior

Error:

error: ';' expected
double precio = 100.0
^

Solución: Agregar ; al final de cada sentencia.

Error 2: Punto y coma en lugar equivocado

double; precio = 100.0; // ✗ ; después de tipo

Error:

error: <identifier> expected

Correcto:

double precio = 100.0; // ✓ ; al final de la sentencia

Error 3: Múltiples puntos y coma

double precio = 100.0;; // ✗ Doble ;;

Esto compila sin error, pero el segundo ; se interpreta como una sentencia vacía (válida pero innecesaria).

Mejor práctica: Un solo ; por sentencia.

Error 4: Punto y coma después de llaves de bloque

public class Ejemplo {
    public static void main(String[] args) {
        double valor = 100.0;
    }; // ✗ NO poner ; después de }
}

Las llaves } NO llevan punto y coma.

Correcto:

public class Ejemplo {
    public static void main(String[] args) {
        double valor = 100.0;
    } // ✓ Sin ;
}

Sentencias que NO llevan punto y coma

Declaraciones de clase:

public class MiClase { // ✓ Sin ;
    // ...
}

Declaraciones de método:

public static void main(String[] args) { // ✓ Sin ;
    // ...
}

Llaves de cierre:

{
    double x = 10.0;
} // ✓ Sin ;

Regla simple: Solo las sentencias ejecutables llevan ;, no las declaraciones de bloques.

Ejemplo: Calculadora de salario con deducciones

public class CalculadoraSalarioNeto {

    public static void main(String[] args) {

        // Sentencia: mostrar título
        System.out.println("===== CÁLCULO DE SALARIO NETO =====");

        // Sentencia: declarar salario bruto
        double salarioBruto = 18000.00;

        // Sentencia: declarar tasa de seguro social
        double tasaSeguroSocial = 3.5;

        // Sentencia: calcular deducción seguro social
        double deduccionSeguroSocial = salarioBruto * tasaSeguroSocial / 100.0;

        // Sentencia: declarar tasa de impuesto
        double tasaImpuestoRenta = 10.0;

        // Sentencia: calcular deducción impuesto
        double deduccionImpuesto = salarioBruto * tasaImpuestoRenta / 100.0;

        // Sentencia: calcular total deducciones
        double totalDeducciones = deduccionSeguroSocial + deduccionImpuesto;

        // Sentencia: calcular salario neto
        double salarioNeto = salarioBruto - totalDeducciones;

        // Sentencias: mostrar desglose
        System.out.println("Salario bruto: Lps " + salarioBruto);
        System.out.println("----------------------------------");
        System.out.println("Deducciones:");
        System.out.println("  Seguro Social: Lps " + deduccionSeguroSocial);
        System.out.println("  Impuesto Renta: Lps " + deduccionImpuesto);
        System.out.println("  Total: Lps " + totalDeducciones);
        System.out.println("==================================");
        System.out.println("Salario neto: Lps " + salarioNeto);

    }

}

Cada línea ejecutable termina con ;

Salida:

===== CÁLCULO DE SALARIO NETO =====
Salario bruto: Lps 18000.0
----------------------------------
Deducciones:
  Seguro Social: Lps 630.0
  Impuesto Renta: Lps 1800.0
  Total: Lps 2430.0
==================================
Salario neto: Lps 15570.0

Sentencias en múltiples líneas

Una sentencia puede ocupar varias líneas por legibilidad:

double totalCompra = precioProducto1 +
                     precioProducto2 +
                     precioProducto3; // ✓ ; al final

El compilador sigue leyendo hasta encontrar el ;

Otro ejemplo:

System.out.println("El precio total es: Lps " +
                   precioTotal +
                   " con descuento aplicado"); // ✓ ; al final

Una sentencia por línea (buena práctica)

Aunque técnicamente puede poner múltiples sentencias en una línea:

double x = 10; double y = 20; double z = x + y; // Válido pero no recomendado

La práctica recomendada es: una sentencia por línea

double x = 10;
double y = 20;
double z = x + y;

Esto hace el código más legible y fácil de depurar.

Sentencias vacías

Un ; solo crea una sentencia vacía (válida pero inútil):

; // Sentencia vacía

No hace nada, pero compila sin error.

Ejemplo con sentencia vacía accidental:

double precio = 100.0;; // Segundo ; es sentencia vacía

Compila, pero el segundo ; no hace nada.

Ejemplo: Sistema de conversión de moneda

public class ConversionMoneda {

    public static void main(String[] args) {

        System.out.println("===== CONVERSIÓN DE MONEDA =====");

        // Cantidad en dólares
        double cantidadDolares = 500.00;

        // Tasa de cambio (1 USD = 24.50 Lps)
        double tasaCambio = 24.50;

        // Calcular equivalente en lempiras
        double cantidadLempiras = cantidadDolares * tasaCambio;

        // Comisión por conversión (2%)
        double porcentajeComision = 2.0;
        double montoComision = cantidadLempiras * porcentajeComision / 100.0;

        // Monto final a recibir
        double montoFinal = cantidadLempiras - montoComision;

        // Mostrar resultados
        System.out.println("Cantidad en USD: $" + cantidadDolares);
        System.out.println("Tasa de cambio: " + tasaCambio + " Lps/USD");
        System.out.println("--------------------------------");
        System.out.println("Equivalente en Lps: Lps " + cantidadLempiras);
        System.out.println("Comisión (2%): Lps " + montoComision);
        System.out.println("================================");
        System.out.println("Monto a recibir: Lps " + montoFinal);

    }

}

Cada sentencia ejecutable termina con ;

Salida:

===== CONVERSIÓN DE MONEDA =====
Cantidad en USD: $500.0
Tasa de cambio: 24.5 Lps/USD
--------------------------------
Equivalente en Lps: Lps 12250.0
Comisión (2%): Lps 245.0
================================
Monto a recibir: Lps 12005.0

Verificar puntos y coma en su código

Checklist:

  1. ✓ ¿Cada declaración de variable termina con ;?
  2. ✓ ¿Cada cálculo termina con ;?
  3. ✓ ¿Cada System.out.println termina con ;?
  4. ✓ ¿Las llaves } NO tienen ;?
  5. ✓ ¿La declaración de clase NO tiene ;?
  6. ✓ ¿La declaración de método NO tiene ;?

Buenas prácticas con sentencias

1. Una sentencia por línea

// ✓ Recomendado
double precio = 100.0;
double descuento = 10.0;
double total = precio - descuento;

// ✗ No recomendado
double precio = 100.0; double descuento = 10.0; double total = precio - descuento;

2. Terminar SIEMPRE con punto y coma

No confíe en que el IDE lo agregue automáticamente. Desarrolle el hábito de escribirlo.

3. No poner ; después de llaves

public static void main(String[] args) {
    // código
} // ✓ Sin ;

4. Dividir sentencias largas en múltiples líneas

// ✓ Más legible
double totalImpuesto = (precioProducto1 + precioProducto2 +
                        precioProducto3) * tasaImpuesto / 100.0;

Ejemplo final: Calculadora de costo de envío

public class CalculadoraCostoEnvio {

    public static void main(String[] args) {

        System.out.println("===== CALCULADORA DE COSTO DE ENVÍO =====");

        // Peso del paquete en kilogramos
        double pesoPaquete = 5.5;

        // Tarifa base por kilogramo
        double tarifaPorKilo = 45.00;

        // Calcular costo base
        double costoBase = pesoPaquete * tarifaPorKilo;

        // Cargo adicional por seguro (5% del costo base)
        double porcentajeSeguro = 5.0;
        double cargoSeguro = costoBase * porcentajeSeguro / 100.0;

        // Cargo fijo de manejo
        double cargoManejo = 75.00;

        // Calcular costo total
        double costoTotal = costoBase + cargoSeguro + cargoManejo;

        // Mostrar desglose
        System.out.println("Peso del paquete: " + pesoPaquete + " kg");
        System.out.println("Tarifa por kg: Lps " + tarifaPorKilo);
        System.out.println("------------------------------------------");
        System.out.println("Costo base: Lps " + costoBase);
        System.out.println("Seguro (5%): Lps " + cargoSeguro);
        System.out.println("Cargo de manejo: Lps " + cargoManejo);
        System.out.println("==========================================");
        System.out.println("COSTO TOTAL: Lps " + costoTotal);

    }

}

Observaciones:

  • ✓ Cada sentencia termina con ;
  • ✓ Una sentencia por línea
  • ✓ Las llaves NO tienen ;
  • ✓ Código claro y legible

Las sentencias son las instrucciones individuales que componen un programa Java, y cada una debe terminar obligatoriamente con punto y coma ;. Este terminador indica al compilador dónde finaliza cada instrucción, permitiendo delimitar claramente las acciones del programa. Desarrollar el hábito de terminar cada sentencia con punto y coma y verificar que las llaves de bloque no lo lleven es fundamental para escribir código Java sintácticamente correcto.

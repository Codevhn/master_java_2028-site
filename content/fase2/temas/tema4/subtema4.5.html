4.5 El método main como punto de entrada del sistema

El método main es el lugar más importante de todo programa Java ejecutable. Es el punto de partida donde la JVM comienza a ejecutar nuestro código, el primer método que se invoca cuando ejecutamos un programa. Sin el método main, un programa Java no puede ejecutarse como aplicación independiente. Comprender qué es el método main, por qué es necesario, cómo debe declararse exactamente, y qué papel juega en la ejecución del programa es fundamental para escribir aplicaciones Java funcionales.

¿Qué es el método main?

El método main es:

El punto de entrada del programa - Donde la JVM comienza a ejecutar el código

Una firma específica obligatoria - Debe escribirse exactamente de una forma determinada

El contenedor de las instrucciones iniciales - Las primeras instrucciones que ejecuta el programa

Analogía:

Piense en el método main como la puerta principal de una casa:

  • Es el punto de entrada obligatorio
  • Todo lo demás se accede desde ahí
  • Sin puerta principal, no se puede entrar a la casa
  • Sin método main, no se puede ejecutar el programa

La firma del método main

Esta es la firma exacta que DEBE tener el método main:

public static void main(String[] args)

Cada palabra es importante y obligatoria. No puede cambiar nada.

Ejemplo completo:

public class MiPrograma {
    public static void main(String[] args) {
        System.out.println("El programa comienza aquí");
    }
}

Cuando ejecuta:

java MiPrograma

La JVM:

  1. Carga la clase MiPrograma
  2. Busca el método main con la firma exacta
  3. Ejecuta las instrucciones dentro de main
  4. Termina cuando main finaliza

¿Qué sucede sin método main?

Programa sin método main:

public class ProgramaSinMain {
    // No hay método main
}

Intento de ejecutar:

javac ProgramaSinMain.java  # ✓ Compila bien
java ProgramaSinMain        # ✗ Error en ejecución

Error:

Error: Main method not found in class ProgramaSinMain, please define the main method as:
public static void main(String[] args)

La JVM es clara: necesita el método main para poder ejecutar.

¿Qué sucede si el método main está mal escrito?

Error 1: Falta public

public class Ejemplo {
    static void main(String[] args) { // ✗ Falta "public"
        System.out.println("Error");
    }
}

Error: JVM no encuentra el main

Error 2: Falta static

public class Ejemplo {
    public void main(String[] args) { // ✗ Falta "static"
        System.out.println("Error");
    }
}

Error: JVM no encuentra el main

Error 3: Tipo de retorno incorrecto

public class Ejemplo {
    public static int main(String[] args) { // ✗ Debe ser "void"
        System.out.println("Error");
        return 0;
    }
}

Error: JVM no encuentra el main

Error 4: Nombre incorrecto

public class Ejemplo {
    public static void Main(String[] args) { // ✗ "Main" con mayúscula
        System.out.println("Error");
    }
}

Error: JVM busca main (minúscula), no Main

Regla absoluta: El método main debe ser EXACTAMENTE:

public static void main(String[] args)

No puede cambiar ninguna palabra.

El método main contiene las instrucciones del programa

Todo lo que escribimos entre las llaves { } del método main es lo que el programa ejecutará:

public class CalculadoraSimple {
    public static void main(String[] args) {
        // ← El programa comienza aquí

        double numero1 = 100.0;
        double numero2 = 50.0;
        double resultado = numero1 + numero2;

        System.out.println("Resultado: " + resultado);

        // ← El programa termina aquí
    }
}

Orden de ejecución:

Las instrucciones se ejecutan de arriba hacia abajo, una por una:

public static void main(String[] args) {
    System.out.println("1. Esta línea se ejecuta primero");
    System.out.println("2. Esta línea se ejecuta segunda");
    System.out.println("3. Esta línea se ejecuta tercera");
}

Salida:

1. Esta línea se ejecuta primero
2. Esta línea se ejecuta segunda
3. Esta línea se ejecuta tercera

Ejemplo: Calculadora de costo total

Archivo: CalculadoraCostoTotal.java

public class CalculadoraCostoTotal {
    public static void main(String[] args) {
        // Inicio del programa (punto de entrada)

        System.out.println("=== CALCULADORA DE COSTO TOTAL ===");
        System.out.println();

        // Datos de productos
        double precioProducto1 = 125.00;
        double precioProducto2 = 280.00;
        double precioProducto3 = 95.50;

        // Calcular subtotal
        double subtotal = precioProducto1 + precioProducto2 + precioProducto3;

        // Calcular envío (10% del subtotal)
        double costoEnvio = subtotal * 10.0 / 100.0;

        // Calcular total
        double costoTotal = subtotal + costoEnvio;

        // Mostrar resultados
        System.out.println("Producto 1: Lps " + precioProducto1);
        System.out.println("Producto 2: Lps " + precioProducto2);
        System.out.println("Producto 3: Lps " + precioProducto3);
        System.out.println("----------------------------------");
        System.out.println("Subtotal: Lps " + subtotal);
        System.out.println("Envío (10%): Lps " + costoEnvio);
        System.out.println("==================================");
        System.out.println("TOTAL: Lps " + costoTotal);

        // Fin del programa
    }
}

Ejecución:

javac CalculadoraCostoTotal.java
java CalculadoraCostoTotal

Salida:

=== CALCULADORA DE COSTO TOTAL ===

Producto 1: Lps 125.0
Producto 2: Lps 280.0
Producto 3: Lps 95.5
----------------------------------
Subtotal: Lps 500.5
Envío (10%): Lps 50.05
==================================
TOTAL: Lps 550.55

Flujo de ejecución:

1. JVM ejecuta: java CalculadoraCostoTotal
↓
2. JVM carga la clase CalculadoraCostoTotal
↓
3. JVM busca método main
↓
4. JVM encuentra: public static void main(String[] args)
↓
5. JVM ejecuta instrucciones dentro de main, línea por línea
↓
6. Cuando main termina, programa termina

El parámetro String[] args

La firma completa del main incluye:

public static void main(String[] args)

El parámetro String[] args permite recibir argumentos de línea de comandos.

Por ahora, necesita saber:

  • Es parte obligatoria de la firma
  • Debe escribirse exactamente así
  • Puede llamarse args o cualquier otro nombre, pero por convención se usa args

Ejemplo con nombre diferente (válido pero no recomendado):

public static void main(String[] parametros) { // ✓ Funciona
    // ...
}
public static void main(String[] argumentos) { // ✓ Funciona
    // ...
}

Pero la convención universal es:

public static void main(String[] args) { // ✓ Recomendado
    // ...
}

Ejemplo: Sistema de registro de venta

Archivo: RegistroVenta.java

public class RegistroVenta {
    public static void main(String[] args) {
        // Punto de entrada del sistema de registro

        // Datos de la venta
        String nombreCliente = "Maria Gonzalez";
        String productoVendido = "Laptop HP";
        int cantidadVendida = 2;
        double precioUnitario = 15000.00;

        // Calcular monto total
        double montoTotal = cantidadVendida * precioUnitario;

        // Calcular descuento por cantidad (5% si compra 2 o más)
        double porcentajeDescuento = 5.0;
        double montoDescuento = montoTotal * porcentajeDescuento / 100.0;

        // Calcular monto final
        double montoFinal = montoTotal - montoDescuento;

        // Registrar y mostrar la venta
        System.out.println("========== REGISTRO DE VENTA ==========");
        System.out.println("Cliente: " + nombreCliente);
        System.out.println("Producto: " + productoVendido);
        System.out.println("Cantidad: " + cantidadVendida + " unidades");
        System.out.println("Precio unitario: Lps " + precioUnitario);
        System.out.println("---------------------------------------");
        System.out.println("Subtotal: Lps " + montoTotal);
        System.out.println("Descuento (5%): Lps " + montoDescuento);
        System.out.println("=======================================");
        System.out.println("TOTAL A PAGAR: Lps " + montoFinal);
        System.out.println();
        System.out.println("Venta registrada exitosamente.");
    }
}

Salida:

========== REGISTRO DE VENTA ==========
Cliente: Maria Gonzalez
Producto: Laptop HP
Cantidad: 2 unidades
Precio unitario: Lps 15000.0
---------------------------------------
Subtotal: Lps 30000.0
Descuento (5%): Lps 1500.0
=======================================
TOTAL A PAGAR: Lps 28500.0

Venta registrada exitosamente.

Múltiples instrucciones en el método main

Puede escribir tantas instrucciones como necesite dentro del método main:

public class EjemploMultiplesInstrucciones {
    public static void main(String[] args) {
        // Instrucción 1
        System.out.println("Primera instrucción");

        // Instrucción 2
        double valor1 = 100.0;

        // Instrucción 3
        double valor2 = 200.0;

        // Instrucción 4
        double suma = valor1 + valor2;

        // Instrucción 5
        System.out.println("Suma: " + suma);

        // Instrucción 6
        double promedio = suma / 2.0;

        // Instrucción 7
        System.out.println("Promedio: " + promedio);
    }
}

Todas las instrucciones se ejecutan en orden, de arriba hacia abajo.

El método main es el contenedor principal

Por ahora, TODO el código ejecutable va dentro del método main:

public class MiPrograma {

    public static void main(String[] args) {
        // ← TODO el código va aquí dentro

        // Declaraciones de variables
        double precio = 100.0;
        double descuento = 10.0;

        // Cálculos
        double precioFinal = precio - descuento;

        // Salida
        System.out.println("Precio final: " + precioFinal);

        // ← TODO el código está dentro de main
    }

}

No podemos poner código fuera del método main:

public class Ejemplo {

    System.out.println("Esto está fuera de main"); // ✗ ERROR

    public static void main(String[] args) {
        System.out.println("Esto está dentro de main"); // ✓ OK
    }
}

Ejemplo: Calculadora de consumo eléctrico

Archivo: CalculadoraConsumoElectrico.java

public class CalculadoraConsumoElectrico {
    public static void main(String[] args) {
        // Punto de entrada del calculador de consumo

        System.out.println("=== CALCULADORA DE CONSUMO ELÉCTRICO ===");
        System.out.println();

        // Lectura anterior del medidor
        double lecturaAnterior = 1520.0;

        // Lectura actual del medidor
        double lecturaActual = 1680.0;

        // Calcular consumo en kWh
        double consumoKwh = lecturaActual - lecturaAnterior;

        // Tarifa por kWh en Lempiras
        double tarifaPorKwh = 3.50;

        // Calcular monto a pagar
        double montoPagar = consumoKwh * tarifaPorKwh;

        // Cargo fijo mensual
        double cargoFijo = 85.00;

        // Calcular total
        double totalFactura = montoPagar + cargoFijo;

        // Mostrar desglose
        System.out.println("Lectura anterior: " + lecturaAnterior + " kWh");
        System.out.println("Lectura actual: " + lecturaActual + " kWh");
        System.out.println("Consumo del mes: " + consumoKwh + " kWh");
        System.out.println();
        System.out.println("Tarifa: Lps " + tarifaPorKwh + " por kWh");
        System.out.println("Consumo: Lps " + montoPagar);
        System.out.println("Cargo fijo: Lps " + cargoFijo);
        System.out.println("========================================");
        System.out.println("TOTAL A PAGAR: Lps " + totalFactura);
    }
}

Salida:

Errores comunes con el método main

Error 1: Olvidar la palabra static

public class Ejemplo {
    public void main(String[] args) { // ✗ Falta static
        System.out.println("No funcionará");
    }
}

Error en ejecución: JVM no encuentra el main

Error 2: Escribir Main con mayúscula

public class Ejemplo {
    public static void Main(String[] args) { // ✗ "Main" en vez de "main"
        System.out.println("No funcionará");
    }
}

Error: JVM busca main (minúscula)

Error 3: Olvidar el parámetro String[] args

public class Ejemplo {
    public static void main() { // ✗ Falta String[] args
        System.out.println("No funcionará");
    }
}

Error: JVM no encuentra el main con la firma correcta

Error 4: Poner código fuera del método main

public class Ejemplo {

    double precio = 100.0; // ✓ Esto es válido (variable de clase)

    System.out.println("Hola"); // ✗ ERROR: instrucción fuera de método

    public static void main(String[] args) {
        System.out.println("Correcto");
    }
}

Las instrucciones ejecutables DEBEN estar dentro de un método.

Plantilla del método main

Use esta plantilla para todos sus programas:

public class NombrePrograma {
    public static void main(String[] args) {
        // Su código aquí

    }
}

Copie esta estructura exacta cada vez que cree un nuevo programa.

Ejemplo final: Sistema de cálculo de comisión

Archivo: CalculadoraComision.java

public class CalculadoraComision {
    public static void main(String[] args) {
        // Sistema de cálculo de comisión para vendedores

        // Datos del vendedor
        String nombreVendedor = "Carlos Martinez";
        double ventasTotales = 45000.00;

        // Porcentaje de comisión
        double porcentajeComision = 8.0;

        // Calcular comisión
        double montoComision = ventasTotales * porcentajeComision / 100.0;

        // Salario base
        double salarioBase = 8000.00;

        // Calcular salario total
        double salarioTotal = salarioBase + montoComision;

        // Mostrar desglose
        System.out.println("========== CÁLCULO DE COMISIÓN ==========");
        System.out.println("Vendedor: " + nombreVendedor);
        System.out.println("Ventas totales: Lps " + ventasTotales);
        System.out.println("Comisión (" + porcentajeComision + "%): Lps " + montoComision);
        System.out.println("-----------------------------------------");
        System.out.println("Salario base: Lps " + salarioBase);
        System.out.println("Comisión: Lps " + montoComision);
        System.out.println("=========================================");
        System.out.println("SALARIO TOTAL: Lps " + salarioTotal);
    }
}

Salida:

Ventas totales: Lps 45000.0
Comisión (8.0%): Lps 3600.0
-----------------------------------------
Salario base: Lps 8000.0
Comisión: Lps 3600.0

SALARIO TOTAL: Lps 11600.0

El método main es el corazón de todo programa Java ejecutable. Es donde la JVM comienza a ejecutar nuestro código y donde colocamos todas las instrucciones que queremos que el programa realice. Su firma debe ser exacta, sin ninguna variación, y todo el código ejecutable debe estar dentro de sus llaves. Comprender el método main es comprender cómo funcionan los programas Java a nivel fundamental.
